# Downgraded Bank

Stage -0.5 ACCEPT but Stage 0.5 verdict falls below the requested
`novelty_target`. The math is sound; the contribution is real but smaller
than proposed.

## Why this bank exists

Negative results, burned seeds, and reusable infrastructure compound over
time. A downgraded entry typically yields:

- a **burned-seed manifest** — specific one-liners refuted by the
  derivation, which the next proposal run on the same anchor should skip;
- **reusable artifacts** — LP setup, response-type enumeration, observation
  operator, literature map, counterexamples — liftable into a different
  angle of the same topic without re-derivation;
- a **proof-attempt epitaph** — 2–3 sentences explaining what collapsed
  and what remains, so the next run does not re-tread the same wall.

## Per-entry metadata

See `../README.md` for the canonical schema. For downgraded entries, the
`gap_reasons[]`, `reusable_artifacts[]`, and `seeds_burned[]` fields are
mandatory — that is the whole point of banking here vs. deleting.
