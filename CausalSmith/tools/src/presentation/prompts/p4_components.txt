A paper's formal environment has no single standalone Lean declaration; its content is
formalized by a COMBINATION of Lean pieces. Identify exactly which pieces, so the website
can show the actual Lean code behind the statement.

Environment body (the paper statement):
{{env_body}}

Available Lean material:
1. Declarations presented elsewhere in the paper (name : file):
{{decl_list}}

2. Theorem statements with their full hypothesis ledgers:
{{theorem_statements}}

Output exactly one JSON object, nothing else:
{"components": [
  {"type": "decl", "decl": "<declaration name from list 1>"},
  {"type": "hypotheses", "theorem": "<theorem name>", "binders": ["<binder name>", ...]}
]}

Rules:
- List every Lean piece that carries part of the environment's mathematical content; omit
  pieces that do not.
- Binder names are the parenthesized hypothesis names exactly as they appear in the
  theorem statement (e.g. _h_Y_lo, _h_consistency).
- If genuinely nothing in the material formalizes the statement, output {"components": []}.
