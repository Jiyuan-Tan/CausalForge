2. Emit ONLY a small JSON manifest that REFERENCES those staged files by `from` (relative to the staging dir). It contains no code — just short paths and one short anchor line per merge, so it always parses.

`{ "ops": [ … ], "notes": "<brief rationale>" }`. `target` paths are RELATIVE to the Causalean root. Op kinds:

- `{ "kind": "create_file", "target": "Causalean/…/Foo.lean", "from": "foo.lean", "newModule": "Causalean.….Foo" }` — write the staged `from` file as a NEW file at `target`. Set `newModule` (the Lean module name) ONLY for a new Lean module, so it is root-wired into `Causalean.lean`. Omit `newModule` for a new non-Lean record file.
- `{ "kind": "merge_lean", "target": "Causalean/…/X.lean", "anchor": "<a line already in X.lean>", "from": "merge_x.lean" }` — INSERT the staged `from` content after the first line containing `anchor` (empty anchor → append at end of file). INSERT-ONLY: the system preserves every existing byte and ROLLS BACK if a merge would change existing content. To add an import to an existing file, use a separate `merge_lean` anchored on an existing `import …` line (its `from` file holds just the import line). Multiple `merge_lean` ops may target the same file. If dependent inserts share one file/anchor, combine them into one staged file in dependency order, or anchor each later insert on text introduced by the prior one. Pick a SHORT, UNIQUE anchor line (a section header, a decl name) so it is unambiguous and easy to escape in JSON.
- `{ "kind": "write_file", "target": "doc/library_review/<Area>.json", "from": "area.json" }` — full-content write of a NON-Lean record file (headline sidecar, `doc/API.md`). Read the current file first, write the COMPLETE updated content to the staged `from` file (add entries/markers, never drop existing ones), and reference it here.
