=== ANCHOR CLUSTER (read first) ===

Identify the anchor cluster of the note from the run's cluster (in the brief / core):
- **Panel**: tuple is `(qid, specialization, p, assignment, target, regression)`.
- **Exact identification (ExactID)**: tuple is `(qid, specialization, observed-law/model class, causal target [effect OR graph/order/mechanism], identifying functional OR structural recovery map, identifying assumptions, observational-equivalence convention)`.
- **Partial identification (PartialID)**: tuple is `(qid, specialization, observable distribution P, parameter θ, identifying restrictions, bounding functional)`.
- **Stat (estimation and inference theory)**: tuple is `(qid, specialization, statistical model + causal estimand θ, sampling model, target object [rate / efficiency bound / limit law / coverage theorem], estimator)`.
- **SCM (graphical identification / partial identification)**: tuple is `(qid, specialization, DAG / ADMG / selection diagram, causal query [interventional / counterfactual / path-specific], structural / identifying assumptions [monotonicity / shape / proxies / selection / extra experiments], identifying functional OR bounding functional)`.

Review the note in the conventions of its cluster. Do NOT demand panel-style FWL / fixed-effect / contamination structure from an ExactID / PartialID / Stat note, and do NOT demand identifying-functional language from a panel note. The estimand-functional node is cluster-aware: for Panel it carries the regression formula (FWL representation), for ExactID it carries either an effect-identifying functional OR a recovery map from the observational law to a structural target (graph, order, edge, mechanism class), for PartialID the bounding functional, and for Stat the target statistical object (rate / efficiency bound / limit law / coverage theorem) together with the estimator. **Cluster by the CAUSAL TARGET, not by the proof technique:** a theorem identifying/recovering a structural direction remains ExactID when Chernoff information, minimax testing, LAN, concentration, or error exponents quantify recovery near a nonidentifiable boundary. Do NOT re-anchor such a note to Stat merely because it proves a recovery-risk limit; Stat is the anchor when the causal target is already identified and the focal object is inference for an estimator of that given target. For a Stat note, identification of θ is GIVEN (do not flag it as unproven); the reviewable content is the estimator construction and the rate / efficiency / limit / coverage claim, which must ADVANCE THE FRONTIER in ONE of the rubric's recognized ways (the (a)–(f) menu in the Frontier-advance floor) — a matching minimax lower bound is NOT mandatory; a sharper rate, a first nontrivial bound, an efficiency-bound-with-attainer, a coverage-claim-with-limit-law, or a novel-estimand estimator each clear the Stat floor. Only a lone bound that is none of these is a Stat-floor miss — see the rubric.

=== DIMENSION: NOVELTY AND POSITIONING ===

**Comparator source status.** Compare public theorem text (article or preprint) normally. Route
published-but-inaccessible load-bearing claims to source verification. Disclose unpublished
non-archival WIP with no theorem text, but do not let it alone trigger a novelty finding or block
the floor.

Check:
- The Introduction states why the problem matters, what is known, what is new, and why the theorem is worth proving / publishing as a math contribution. Do NOT require Lean-formalization motivation — Stage 1 owns the formalization gate.
- Related work is targeted and sufficient: every cited paper or accepted-bank in-repository artifact has a relevance sentence.
- The note compares the main theorem with the closest known result classes and states precisely what differs.
- Novelty claims are theorem-level, not generic topic-level claims.
- Main theorems are genuine contributions of the note, not incremental lemmas, definition restatements, or sanity checks.
- After each main theorem, the note explains what is new, why it matters, and how it compares with the closest literature or accepted-bank in-repository prior result.
- The note does not claim an exhaustive literature review unless the brief supplies enough evidence.
- The note does not keep a separate "Bibliographic note" as a substitute for substantive related-work positioning.

Negative-result / counterexample novelty (load-bearing):
If the flagship is a counterexample, impossibility, sharpness-failure, or "method X is not valid" claim, the note must (a) name a specific published paper / estimator / workflow that uses the refuted method (bibkey + claim — generic "natural shortcut" framing is not a target); and (b) either witness the obstruction on a non-degenerate observable law, or promote the result to a generic-class statement (open set of observable laws / proxy systems / graphs, not one hand-tuned rational instance with Y constant a.s. or the constraint geometry engineered backwards). Failure of (a) → `revise` with classification `novelty`; failure of (b) only → cap the tier-awareness reasoning at `subfield`. Do not `pass` at `field` tier-awareness if either fails.

=== DIMENSION: DELIVERABLE / USABILITY GATES ===

Proof-level correctness — algebra, decompositions, bias/variance terms,
orthogonality claims, rate balances, matching converses, numerical witnesses — is
owned by the parallel math referee (combined worst-of); do NOT re-derive it here.
The checks below are DELIVERABLE / usability gates that YOU own (they concern the
SHAPE of the conclusion, not the proof's correctness):

- The causal estimand and the estimand-functional node remain distinct until proved equal (Panel, ExactID) or shown to bracket the target (PartialID); the note must not define the target so as to make the theorem tautological; no theorem conclusion is just a hypothesis or definition-body restatement.
- **Usability of the conclusion (UNIVERSAL).** Classify the headline theorem by type, then apply the matching deliverable test:
  - *Identifying-formula / bound* (Panel coefficient, ExactID identification, PartialID bound): the estimand-functional node and the theorem node must expose a computable map from observed-population inputs. The gate is *evaluability*, not any specific formalism — allow whatever form the author chose as long as a downstream reader can evaluate it on P. Examples that satisfy the gate (illustrative, not a closed list): closed form, CDF / quantile / copula envelope, extremal-coupling / optimal-transport representation, support-function / random-set kernel, explicit algorithm with stated complexity, named optimization with explicit dimension/constraints, Tikhonov-regularized inverse / completeness-bridge formula with stated regularization, formula in evaluable named operators. Do NOT flag a usable form merely because it is not on this list. NOT sufficient: bare inf/sup of an unspecified feasible set; "θ is identified" without the formula; a Panel contamination term named but not written; an ExactID functional with an opaque inverse and no algorithm. *Flagship floor (math-shape only — not empirical):* for flagship-aspirant theorems, the theorem must single out a *named* focal mathematical object that the theorem identifies / characterizes / constructs. The object can take many forms (see the object-type menu in the injected FLAGSHIP RUBRIC's named-focal-object floor) — the requirement is that the theorem name it, not that it be of any particular type. A theorem of the form "Θ_I(P) is the image of an LP / random-set / copula family" with no named object singled out as the contribution passes the universal usability gate but does NOT clear the flagship floor; tier caps at field. This is a math-content requirement; the pipeline does not run experiments.
    - Failure A (content absent): no evaluation map exists. Flag as a deliverable-gate `revise`.
    - Failure B (content buried): the computable content exists in `proof_tex` or the counterexample node but the estimand-functional node / theorem node still read as inf/sup of an unspecified feasible set. Route as the appropriate node finding — lift the content.
  - *Counterexample / negative result*: the theorem node or counterexample node must produce an explicit witness — a concrete observable law on which the failure is quantified, or a generic-class certificate (generic region of laws / proxy systems / graph family — non-degenerate, not a single instance). This is already a Dimension 2 check; flag here only if the witness is asserted but not exhibited.
  - *Equivalence / characterization*: at least one side of the equivalence must be a computable map, OR both sides must be named objects with prior known evaluation. An equivalence between two opaque optimizations fails this gate; flag as a deliverable-gate `revise`.
  - *Sensitivity / robustness*: if the headline statement is a sensitivity result (e.g. "the bound widens under δ-deviation from parallel trends", "the identified set inflates under ε-violation of monotonicity"), the estimand-functional node / theorem node MUST expose (i) closed-form or algorithmic bounds `L(δ)`, `U(δ)` as functions of the violation parameter, AND (ii) at least one *critical-δ* quantity defined as a math object — the smallest δ at which a sign flips, an estimator ranking inverts, 0 enters the bound, or a named identifying claim collapses. The critical-δ is a math content requirement, not an empirical-validation one: the theorem must single it out by definition, but the pipeline does not need to evaluate it on data. Top-journal sensitivity papers (Rambachan-Roth ReStud 2023, Adão-Kolesár-Morales QJE 2019, Mogstad-Santos-Torgovitsky AER 2018) define such critical-δ objects formally; a sensitivity result that merely says "the bound widens" without the closed-form δ-trajectory and a named critical-δ fails the gate. Flag as a deliverable-gate `revise` if the trajectory is absent; route as the appropriate node finding if the trajectory is buried in `proof_tex` but the estimand-functional node / theorem node read as qualitative.
  - *Pure impossibility / generic-class*: the obstruction certificate must be explicit (quantifier elimination, dimension count, topological argument). "By a standard argument" without the certificate fails the gate; flag as a deliverable-gate `revise`.
  - If the theorem does not fit any of the above types, the note itself must say which type it is and what the corresponding deliverable is. A note that asserts no type and produces no deliverable fails the gate.

Cluster-specific checks (confirm the expected sanity-check content is present — the proof algebra is the math referee's job; do not re-derive it here):
- **Panel**: whenever possible, the theorem states `target effect + contamination terms`. Sanity checks reduce to homogeneous effects, no carry-over, vanishing contamination terms, T=2 reducing to canonical DID (or the named known special case).
- **Exact identification**: for effect-ID, proofs cite the relevant do-calculus / SWIG / consistency / ignorability rule and invoke positivity only on used strata. For structure/mechanism-ID, proofs instead expose the observational recovery map, the structural indeterminacy/equivalence convention, and the identifiable-versus-nonidentifiable boundary; they need not cite an effect-identification rule. Sanity checks must match the target (effect-ID examples include X = ∅ or random treatment; structure-ID examples include the reversible/observationally-equivalent center and a named identifiable special case).
- **Partial identification**: bound proofs distinguish "outer bound" from "sharp"; sharpness claims must come with a construction appropriate to the technique used (extreme-point / completion construction; explicit extremal coupling; support-function attainability; capacity-functional realization; copula attainability witness; etc.) or a citation. Sanity checks include the no-restriction Manski bound (vacuous), point identification under maximally strong restrictions (zero-width), the MTR / MTS specialization, and the named special case (e.g. Balke–Pearl reduces to Wald). Do NOT demand an LP framing if the substrate naturally expresses the bound in a non-LP form.

=== VERDICT POLICY ===

Return one of:

- `pass`: novelty/positioning and the deliverable/usability gates both pass. (Proof-level correctness is judged independently by the math referee and combined worst-of — your `pass` is about your own dimensions.)
- `revise`: the note has a real theorem-level contribution but has fixable novelty/positioning, exposition, proof, example, or assumption gaps.
- `fail`: the note has a non-fixable or severe failure: mathematically false main theorem, tautological theorem, non-novel flagship result, misleading literature positioning, unrecoverable proof gap, or anchor-cluster drift that changes the problem. **Exception (refutation-with-positive-result):** if the headline theorem is "mathematically false" but the Stage 0 derivation simultaneously proves a strictly stronger or strictly opposite POSITIVE result that satisfies any flagship axis (a)–(j), do NOT `fail` — use `pass` with flagship tier-awareness per the flagship rubric path (d). This carve-out only applies when the positive result is independently flagship-grade.

Classify non-pass verdicts by the primary failing dimension:

| Classification | Meaning |
|---|---|
| novelty | Claimed result appears non-novel, related work is insufficient, closest prior work is omitted or mischaracterized, or theorem importance/comparison is absent. |
| deliverable | The conclusion's mathematical deliverable is unusable: no evaluation map / computable form, an opaque-vs-opaque equivalence, a missing critical-δ trajectory, or an unexhibited witness. (Proof-level correctness is the math referee's classification, not yours.) |
| mixed | More than one dimension has material failures. |
| other | Failure does not fit these dimensions; explain why. |

Use `revise` rather than `fail` when the likely fix is local or the theorem can plausibly be repaired without changing the core problem. Use `fail` when the flagship claim itself collapses or the artifact is not a viable Stage 0 theorem note.

**Fail discipline (anti-meta-claim).** Fail is for the *current artifact*, not a prediction about the conjecture's eventual repairability. Do NOT include unqualified meta-claims like "no realistic local repair would lift this kernel to flagship" or "no local revision would lift it to flagship" in your findings — those drive the downstream intervention judge into the conjecture-level pivot path (Case 6b: stage_neg1) and burn an angle even when the defect is derivation-level (re-derive territory: stage_0). If you mean "the current derivation is below flagship", say exactly that and let the intervention judge route. If you mean "the conjecture as stated admits a counterexample / is structurally vacuous / has an inconsistent regime", say *that* and cite the witness. The two are different routes; only the second is a true Case 6b.

**Anchor / missing-assumption defects are typically `revise`, not `fail`.** If the kernel correctly proves something but the proven object belongs to a different cluster (panel / exactid / partialid / stat) than the run declares, or a load-bearing assumption is silently omitted from the headline theorem statement, route a node-level revise finding and direct Stage 0 to re-derive under the corrected scope or with the assumption surfaced. Only escalate to `fail` if the corrected scope / surfaced assumption would itself collapse the flagship claim (e.g. the assumption that makes the theorem true is what the proposal claimed to dispense with).

=== TIER-AWARENESS REASONING (NO OUTPUT FIELD) ===

The publishability tier is assessed separately by the cold referee; do NOT emit a tier. Use this rubric only to decide whether the core falls below the injected novelty_target/floor and therefore needs a `novelty` revise. This is the drift signal against the Stage -1 `publishability_tier` (which captured what the *proposal* promised). Apply the rubric strictly — venue prestige and writing quality are NOT inputs.

| Tier | Rubric (what the derivation must demonstrate) |
|---|---|
| `flagship` | A novel main theorem is *proved* (not assumed); the proposal's theorem node AND every solve-target node is either proved or sharply refuted with a stated counterexample; the assumption budget added in the relevant assumption / estimand-functional nodes satisfies the Bucket A allowance defined below (up to 2 NAMED structural Bucket A assumptions tolerated, free Bucket A assumptions still cap at field — see hard rules); AND the *derived* result satisfies at least ONE regime axis (a)–(j) defined in the injected **FLAGSHIP RUBRIC** block (apply its derived-vs-assumed gate here: an axis satisfied only by an assumed quantity in the estimand-functional node does NOT count). |
| `field` | A novel reduction opens a regime the literature had not isolated. This includes Panel with a new spectral/structural step; ExactID with a new effect-identifying functional, graph/independence premise, structural recovery map, identifiable/nonidentifiable frontier, or local information boundary for graph/order/mechanism recovery in a previously untreated model regime; PartialID with a shape/sensitivity condition that strictly tightens prior bounds; and Stat with a new rate/efficiency result whose matching converse is missing, or a proved nonmatching two-sided bracket that advances the frontier. Generic testing, LAN, concentration, or algebra used as the **proof engine** does not by itself demote a new ExactID recovery-map/frontier contribution. A matching minimax result is the strongest form, not the field floor. At least one solve target is proved or sharply refuted. Any labelled key step credited as the novelty must be a condition or a derived result, NOT an assumed construction of the object the headline characterizes; an assumed construction of the answer is a definitional unfold and caps at `subfield`. Acceptable to rely on user-approved Bucket A assumptions if they are genuinely new named conditions. Formalization-readiness is NOT a publishability axis. |
| `subfield` | A useful technical contribution within an established framework: a new lemma, sharper counterexample, or specialization, but not a regime-opening theorem. The core mathematics is essentially well-known machinery in the cluster — generic linear algebra + support count for Panel; a routine application of the back-door / front-door / IV criterion for ExactID; a textbook Manski / Balke–Pearl bound restricted to a special case for PartialID; a standard CLT / delta-method / plug-in rate or a textbook efficiency-bound calculation for Stat. |
| `incremental` | The proved content is generic identity + a counting / support / case-split formula already implicit in the cited literature; the novelty is framing, not new mathematics. Use this when every "novel" theorem node is either (Panel) an inherited row-space / projection lemma or a standard FWL identity, (ExactID) a textbook adjustment formula re-derived under standard assumptions, (PartialID) a Manski / Balke–Pearl bound re-derived under restrictions that already give point identification or already appear in the cited literature, or (Stat) a direct application of a known CLT / concentration inequality / efficiency bound with no new rate, exponent, or matching converse. |

Hard rules:
- **Bucket A allowance at flagship.** Up to 2 user-approved Bucket A assumptions are tolerated at `flagship` IF each one is a **named structural condition** — i.e. the assumption corresponds verbatim or near-verbatim to a published, named identifying restriction (the relevant core node must cite the bibkey + the published name of the condition, e.g. "Assumption 3 [Imbens-Angrist 1994 monotonicity]" or "Assumption 5 [Miao-Geng-Tchetgen Tchetgen 2018 completeness on the outcome bridge function class H]"). A "free" Bucket A — one that re-states the flagship claim under a different label, OR a generic regularity / smoothness / positivity / strict-gap-positivity premise without a published name — does NOT qualify and triggers `field` cap even if alone. More than 2 named Bucket A assumptions OR any single free Bucket A → at most `field`.
- This is not the proposal tier. Do not copy the Stage -1 `publishability_tier`; re-evaluate the actually-derived content.
- Negative-result kernels: apply the negative-result HARD CAPS in the injected FLAGSHIP RUBRIC block (`N-strawman` → `incremental`; `N-witness-trivial` → `subfield`; `flagship` requires a named published target AND a generic-class obstruction).
- **Refuted-conjecture re-tiering (survivors-only).** When any solve-target node is refuted at Stage 0 (explicit counterexample / disproof), assess tier-awareness on the *surviving proved* claims ONLY — a refuted claim is a negative result, scored on its own, and contributes NOTHING toward the positive tier. Credit the survivors normally: if a surviving proved claim is itself a regime-opening kernel (named focal object, new regime / target / object firing an axis (a)–(j)), the tier reflects that survivor (e.g. a sharp-bound kernel that holds while only a secondary inference conjecture is refuted keeps its kernel-driven tier). But if the refuted claim WAS the tier driver, demote to what the remainder earns. In particular (**Stat**): a surviving achievability rate / estimator whose *matching converse was refuted* (not merely absent) caps at `subfield`, UNLESS the surviving upper-rate result independently fires a regime axis (a) / (f) / (g). This is strictly stricter than the field-tier "converse missing" case: a *missing* (never-attempted) converse is the `field`-cap in the tier table; a *refuted* converse is worse and demotes one tier to `subfield`. The same survivors-only logic applies to a refuted iff-direction, a refuted sharpness claim, or a refuted inference conjecture in any cluster — re-tier on what is actually proved, never on what was disproved.
- **Open-kernel re-tiering (unproved load-bearing solve-target).** Sibling of the refuted rule, for the *unproved-but-not-refuted* case. If a load-bearing solve-target is left `partial` because its defining claim is unproven — the theorem node proves only a strictly weaker surrogate (a one-sided / outer bound where the headline claimed sharp, a one-direction-only proof where it claimed iff, an upper rate with no converse AT ALL (lower bound absent or left open), a conditional-on-a-hard-premise version) with the defining content deferred to an OPEN `Conjecture` — tier on the surviving surrogate ALONE; if its interest rode on the open claim (what remains is a setup / conservative device), cap the tier-awareness reasoning at `subfield` and use `kernel_substituted` in reasoning. This fires EVEN WHEN the proposal headline was restated down to the surrogate (anti-laundering — mirrors the statement_correction result-class gate). THREE things are explicitly NOT triggers: (1) the result delivered in its recognized standard form with only an over-precise form relaxed (the standard form IS the contribution — keep its earned tier); (2) a genuinely useful standalone surrogate (tractable, named consumer, firing a regime axis (a)–(j) on its own) keeps its tier; (3) a fully-PROVED two-sided rate bracket — a sharp converse AND a feasible achievability BOTH proved but not coinciding — that advances the frontier (first/sharper bound in a regime prior art lacked): the honest converse↔achievability gap is a quantitative finding, NOT a defining half left open, so keep its earned (field) tier. The trigger is the headline's result class degrading to a weaker one with its defining half left OPEN/deferred — never the relaxation of an over-precise form whose underlying result still holds, and never an honest gap between two PROVED bounds.
- **Open-ended Questions are solve targets (tier parity).** OEQ core nodes are folded into the Stage-0 solve fanout exactly like `Conjecture` nodes — Stage 0 CONSTRUCTS the asked-for object, and the constructed result carries a verdict (confirmed / refuted) through the same kind-agnostic plumbing. Therefore, everywhere this section says "Conjecture", read "Conjecture **or** Open-ended Question (solve target)": a solved OEQ whose constructed object is *proved* counts as a proved main theorem for `flagship` / `field` (subject to the same derived-vs-assumed gate and floors), and a solved OEQ whose candidate construction is *refuted* counts as a refuted conjecture (subject to the refuted-conjecture re-tiering rule above). A note whose only solve targets are OEQs is NOT capped below `field` on that basis alone — tier it on the constructed-and-proved objects, identically to a conjecture-based note.
- Usability-of-conclusion floor: if the headline is an identifying-formula / bound theorem and the estimand-functional node + theorem node expose no evaluation map (in any of the acceptable forms — closed form, CDF / quantile / copula envelope, extremal coupling, support-function / random-set kernel, explicit algorithm, named optimization with explicit dimension and constraints, Tikhonov / bridge-function formula with stated regularization, or formula in evaluable operators), cap the tier-awareness reasoning at `subfield` — a sharpness proof of an unspecified feasible set is a definition restatement, not a regime-opening contribution. For equivalence theorems, the cap also applies if neither side is computable.

**Flagship strengthening rules (soft tier movers).** Apply the soft tier-movers and floors defined in the injected **FLAGSHIP RUBRIC** block (iff bump, joint-identification bonus, comparator-chain / named-focal-object / estimation-duality / concrete-witness / exhibited-object floors, tautology / definitional-unfold / free-parameter screens), AFTER the axis check, and with the floor-before-bump ordering stated there. The tier-awareness cap on each fired floor is `field`.

Stage 0.5 nuance on the iff bump: a direction-only proof (converse missing) does NOT earn the bump and, if the proposal promised iff, is exactly the `direction_only` shortfall in the taxonomy below.

=== PROPOSAL_PROMISE_GAP (REASONING ONLY) ===

When the derived kernel falls short of what the proposal headlined, classify the *shape* of the shortfall in your reasoning. This separates solver-shortfall failures (proposal was flagship-shaped, Stage 0 substituted a weaker derivable claim) from genuine-novelty-shortfall failures (proposal was overrated and the derivation faithfully delivered the modest kernel implied). Use the taxonomy to inform finding text; it is not a required output field.

| Value | When to use |
|---|---|
| `kernel_substituted` | Stage 0 derived a different (weaker) object than the proposal headlined. The proposed Theorem 1 reads as object A; the produced Theorem 1 proves object B with B ⊊ A. Example: proposal headlines distributional sharpness; derivation produces a row-span identity that doesn't characterize the full distribution. |
| `assumption_omitted` | Flagship object derived only after silently dropping a load-bearing assumption named in the proposal's setup / estimand-functional nodes. The theorem statement quietly omits the assumption; without it, the sharpness/identifiability claim doesn't apply to the advertised model. Example: rank-invariance assumption qif present in the estimand-functional node but absent from the headline theorem. |
| `direction_only` | Proposal promised an iff / equivalence frontier / sharp characterization (both directions). Stage 0 proved only one direction (typically soundness, or a counterexample to a strengthened converse). Example: proposal headlines "iff identifiability"; derivation proves "if" + a counterexample to a strengthened "only if". |
| `constructive_object_missing` | Proposal promised an explicit enumeration, polynomial-time algorithm, closed-form threshold formula, or constructive witness; Stage 0 only proved existence, used generic LP/duality, or appealed to non-constructive arguments. Example: proposal headlines "exposed-basis enumeration of the polytope"; derivation argues via LP duality without producing the enumeration. |
| `tier_genuinely_below` | The derived kernel exactly matches the proposal AND that kernel is honestly subfield/field — no solver shortfall. The -0.5 reviewer overrated the novelty potential. Use this when the conjecture was already an algebraic specialization and Stage 0 delivered it faithfully. |
| `proposal_drift` | The proposal *itself* drifted off-topic between Stage -1.2 drafting and Stage 0: its estimand-functional / solve-target nodes and named comparators no longer line up with the cited published open problem the proposal claims to address. Stage 0 derived the solve-target nodes faithfully (per-target verdicts honest), but the conjectures are not the published target. Use this when the per-target split returns at least one `partial`/`confirmed` verdict yet the headlined published comparator (e.g. Greenland/VanderWeele collider envelope, Honoré–Hu bounds) is *named in the proposal abstract* but *not actually established* by any solve-target node. This is a proposal-stage defect, not a solver shortfall — banking should flag `reusable: not_reusable` because the proposal needs re-drafting, not the solver retried. |

Discipline: pick exactly one value, the most specific that fits. If the shortfall is `kernel_substituted` AND `assumption_omitted` (the substituted kernel only holds after dropping an assumption), pick `assumption_omitted` (it's the more diagnosable shortfall). If `direction_only` AND `constructive_object_missing` (the missing direction would have produced the constructive object), pick `constructive_object_missing`. Reserve `tier_genuinely_below` for cases where you can articulate why the *exact* derived kernel is already subfield — not as a catch-all. Pick `proposal_drift` only when the per-target verdicts are honest AND the proposal's headlined published comparator is not derivable from any solve-target node (the proposal itself migrated topics between -1.2 and 0); a single `partial` verdict alone is NOT enough — it must be paired with a missing named-comparator clause.
