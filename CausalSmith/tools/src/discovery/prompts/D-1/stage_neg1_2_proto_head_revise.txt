=== MODE: revise (vN+1, kernel preserved) ===

You are producing the NEXT version of an existing angle. The kernel is preserved; you PATCH only the items the Stage -0.5 reviewer flagged. Do NOT re-ideate (no Step 0/1 seed generation) and do NOT re-run the literature scan — the seeds, motif, and lit work carry over via the brief.

Inputs: the PRIOR PROPOSAL CORE is injected below (the artifact you revise); the PRIOR REVIEWER VERDICT block carries the S / N / C flags to address.

TOP PRIORITY — ORCHESTRATOR DIRECTIVES: when an ORCHESTRATOR ESCALATION LOG is present, every `DIRECTIVE` is additional required work and overrides this mode's reviewer-flags-only / kernel-preserved scope wherever they conflict. Apply it even when the prior verdict is `ACCEPT` with empty S / N / C flags, and make every coordinated same-angle edit needed for the full core to agree. Empty reviewer flags never authorize a no-op in the presence of a directive. If a directive is genuinely impossible, return `needs-pivot` with the mathematical obstruction; never silently retain the prior core.

For each flag, edit the corresponding core field and re-emit the FULL core. The PRIOR REVIEWER VERDICT also carries `strengthening_paths` — consult them for the SUGGESTED REMEDY of a flag (especially a relabel/route option, e.g. "relabel the converse as an open-ended question with the … perturbation route as the Handle"). Apply a strengthening path when it IS the fix for a flagged item, but do NOT force-apply optional sharpening beyond the flagged items (that is scope creep / over-build).
- S (structure) → fix the offending formal item (a missing assumption an unstated statement cites, a mis-typed symbol, a missing bibliography entry).
- N (novelty) → sharpen `related_work` / a statement's `gap`, OR demote a `theorem` to an `openendedquestion` (`oeq:` id plus a concrete Handle) when the verdict says it is not strictly stronger than published prior art. Legacy `conjecture` is accepted only for existing cores; do not introduce it.
- N-thin-anchor / N-comparator-drift / N-thin-survey → these REQUIRE touching the literature (the ONE exception to "do not re-run the literature scan"): OPEN each flagged comparator paper (arXiv `.tex` / PDF). (a) `N-thin-anchor` → add a section/equation/page/quoted-open-problem anchor to each flagship comparator citation. (b) `N-comparator-drift` → add a statement phrased as a strict tightening / reduction against the named published result (e.g. "the frontier reduces to [comparator]'s bounded-overlap rate under strict overlap"), or downgrade "extends" to "informed by" — and in BOTH cases FILL the structured `comparator_promise_table` row for that comparator (set `matched_by` + `match_kind`); SC6 audits the structured table, not the `related_work` prose, so a prose-only edit re-fires the flag. (c) `N-thin-survey` → add the specific missed papers the verdict names, one relevance sentence each. A positioning flag that RECURS across rounds means the prior revise edited prose without opening the papers — do the reading this time, do not just reword.
- N-promissory-object on a CONVERSE / lower bound (e.g. "minimax ≍ R*") → do NOT try to build the full witness here; the explicit construction is the proof's job. PREFER: name the construction route as a Handle (e.g. "matching lower bound via a Le Cam two-point / Assouad family over inverse-weight + tail perturbations") and relabel the converse as an open-ended question. Never patch it with an EXISTENTIAL placeholder ("for any set C_n there exist amplitudes δ …") — that is itself promissory and re-fires the flag. Commit a construction inline ONLY if it is short and fully explicit (named perturbations with stated bounded pairwise divergence).
- C (soundness) → fix the flagged `statement` / `assumption` / `target_estimand` per the verdict; re-run the WELL-POSEDNESS SELF-CHECK over what you touched.

CONSISTENCY-PROPAGATE (the top cause of revise loops that never converge): a patch must make the WHOLE core agree, not just the flagged line. Changing a claim/assumption/symbol means updating or DELETING every place that asserts or depends on the old version (the `tldr`, `project_justification`, dependent statements, `used_by`/`depends_on` edges); an assumption that only existed to license a dropped claim is DELETED, not caveated.

SELF-CONTAINED: the revised core must read as a standalone proposal of the kernel as it now stands — no "previously / we now weaken / the original" changelog prose in any field. `honest_scope` states what is open as a standing fact, never as a record of the edit.

Terminal status: `completed` (you patched the flagged items and the revised core is ready) OR `needs-pivot` (the verdict carries a defect revise cannot fix in place — a definitional-unfold / tautological-iff kernel, an unsalvageable construction). Do NOT self-select kernel-replace or draft-rebuild; the orchestrator dispatches those automatically on a REJECT.
