=== MODE: cold-start (v1 of a new angle) ===

You are authoring the FIRST version of a new proposal angle, from scratch. Run Steps 0 → 1 → 2 to choose the kernel, then emit the typed core per the body below.

Step 0 — Seed generation from named claims. Produce ≥8 candidate seeds, each anchored to a named open problem from the D-1.1 gaps context (a published paper bibkey or a prior-proposal ref) — or a genuinely new object you can defend as unanchored. BEFORE posing a seed, OPEN its anchor: fetch the paper (arXiv `.tex` preferred; PDF / publisher page otherwise) and read the Introduction (the problem it poses) and the Conclusion / Future Work (the open questions the authors themselves flag); a strong seed LEVERAGES one of those open questions, not one the paper already answered. Shape each seed with a motif from the MOTIF LIBRARY block (when one is injected in the brief) and note its ambition (flagship / field / subfield). Red-team each seed in one line — the most cynical referee objection ("immediate from [paper] because …"); drop it if the objection holds, else keep the one-line rebuttal. Score on importance / novelty (name the closest published result and the precise gap) / feasibility / sharpness / leverage, and keep the top 2–3 as candidate angles. If you cannot get ≥6 credible seeds, emit `status:"needs-pivot"` rather than padding with weak ones.

Step 1 — Deepen literature scan on the chosen seed(s). For each surviving candidate: pull 2–4 additional adjacent threads (papers citing / cited by the anchor, the most recent survey), confirm what is settled vs open, and confirm no closer paper subsumes the kernel. Read the key part of the anchor paper to get inspiration. Open the method / writing exemplars the gap entry points to and read the technique you ADAPT (estimator construction, limit-theory proof, identification move) instead of reinventing it; imitate the anchor's setup / motivation arc. Drop a candidate whose novelty rebuttal collapses or that now has fewer than ~3 prior-art candidates. If none survive, emit `status:"needs-pivot"`.

Step 2 — Draft the core JSON. Pick the highest-scoring surviving candidate and author it as the typed proposal core per WHAT TO EMIT + the discipline sections in the body below; run the WELL-POSEDNESS SELF-CHECK before emitting. Emit the cold-start handoff fields on the final stdout line: `seeds`, `seed_details`, `literature_map`, `cluster`, `novelty_justification`, `literature_checklist`.
