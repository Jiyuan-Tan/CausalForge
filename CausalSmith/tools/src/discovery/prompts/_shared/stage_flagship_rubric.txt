=== FLAGSHIP RUBRIC (shared — single source of truth for both reviewers) ===

This block is the canonical statement of the flagship regime axes, the soft tier-mover
floors, and the hard caps. It is injected verbatim into BOTH the Stage -0.5 proposal
reviewer and the Stage 0.5 derivation reviewer so the two stages key off the SAME
rubric. The only thing that differs between the two stages is WHAT is being judged:

- Stage -0.5 judges a PROPOSAL whose Conjectures are intentionally open — the tier is
  HYPOTHETICAL, conditional on every Conjecture resolving favourably. A proposal is
  flagship-eligible iff, assuming favourable resolution, the eventual derivation would
  satisfy at least ONE axis (a)–(j) below.
- Stage 0.5 judges a DERIVATION — the tier is what the proof ACTUALLY delivers. A
  derivation is flagship iff a novel main theorem is *proved* (not assumed) AND the
  delivered content satisfies at least ONE axis (a)–(j) below.

THE DERIVED-VS-ASSUMED GATE (read first — the most common calibration leak). For BOTH
stages: a focal object that is *encoded in a §7 assumption*, or that unfolds the very
definition of the object on the other side of an iff, is NOT a flagship contribution —
it is a definitional restatement. At Stage -0.5 you cannot check the proof, but you CAN
check structure: trace every load-bearing RHS symbol of the headline focal object to a
§6 observable primitive vs a §7 assumed quantity (the SC10 assumption-substitution
screen). If the flagship novelty rides on a §7-assumed quantity, or the iff is a
definitional unfold, cap at field NOW — do not pass it downstream to be discovered at
Stage 0.5. (Past calibration gaps were almost all this shape: "frontier framed but not
derived", "frontier encoded in Assumptions 4+6", "tautological row-stacking at the
population truth".)

=== REGIME AXES (a)–(j) — satisfy at least ONE for flagship ===

(a) **Regime opening.** A previously open identification / estimation regime is resolved
    (new ID result for ExactID; new sharp / strictly-tighter bound for PartialID;
    previously open identification / estimation regime for Panel).
(b) **Equivalence AND non-equivalence frontier between named published estimators.** Both
    an asymptotic-equivalence (same influence function / identified set / efficiency
    bound) AND a strict non-equivalence between ≥2 specific published estimators /
    identification strategies (each cited by bibkey + estimator name), with a
    computable-from-observables threshold (closed-form, explicit algorithm, or named
    optimization with explicit dimension and constraints) sharply separating the two
    regimes — an iff characterization, not one direction. Existence/absence statements
    already in the cited prior art do NOT qualify; the threshold must be a new object.
(c) **Strict extension to a wider class.** The same identifying / bound / efficiency claim
    as a named published result on a class of observable laws / proxy systems / graph
    structures / parameter regimes that *strictly contains* the previously-handled class,
    with the prior literature explicitly leaving the larger class open (cite the
    open-problem statement). Parametric reweighting, notational generalization, change of
    regularization, or re-stating under different sufficient conditions does NOT qualify.
(d) **Sharp refutation of the flagship kernel via a positive result.** The derivation
    cleanly refutes the proposed Theorem/Conjecture (explicit counterexample, contradiction,
    or limit calculation) AND proves a strictly stronger or strictly opposite POSITIVE
    result that itself satisfies another axis. (Stage 0.5: ACCEPT with
    `kernel_status: "refuted_with_positive_result"`; do NOT route to reject / pivot. The
    refutation counts as new math ONLY when the positive result is independently
    flagship-grade.)
(e) **Diagnose-then-correct of a widely-cited estimator.** (i) decompose a widely-used
    published estimator E (bibkey + name) into a weighted sum of structural components;
    (ii) characterize a contamination / bias term Δ that vanishes iff a stated condition C
    holds, naming the regime where E collapses or flips sign; (iii) propose a corrected
    estimand / estimator targeting the clean object with a computable map from observables.
    Exemplars: Goodman-Bacon (JoE'21), Sun-Abraham (JoE'21), Borusyak-Jaravel-Spiess
    (ReStud'24), de Chaisemartin-D'Haultfœuille (AER'20). A pure decomposition (i)+(ii)
    without (iii) may still qualify via axis (a) if the contamination characterization
    opens a previously-closed regime. "Found a bug in E" without a corrected estimand does
    NOT qualify.
(f) **New identification target.** Identify (or sharp-bound) an estimand the field has not
    previously identified. Acceptable without a named comparator IF the target is genuinely
    new; cite ≥1 paper in the same setting that did NOT identify it. Routine specializations
    / notational variants do NOT qualify.
(g) **New mathematical object made causally relevant.** Import or construct an object
    (sharp-set representation, extremal coupling, capacity kernel, info-theoretic quantity,
    geometric invariant, new function class) and prove it characterizes ID / sharpness /
    equivalence / impossibility in a way no prior causal-inference work has. Renaming /
    rewriting a known object does NOT qualify.
(h) **Cross-field bridge.** A non-obvious theorem-level connection between two
    previously-separate frameworks (equivalence / sharp inequality / mutual-characterization,
    not a survey-style overlap). Cite both frameworks' representative papers.
(i) **Algorithmic / constructive identifiability.** An explicit decision procedure for
    identifiability in a class where prior work was non-constructive or open. State input
    format, decision predicate, complexity / termination bound. A bare LP without complexity
    does NOT qualify.
(j) **Design-based / experimental contribution (Experimentation cluster).** One of: a limit law
    + Wald-coverage for an explicit estimator of a causal estimand under interference, DERIVED
    from named §7 primitives (design randomness with fixed potential outcomes — dependency-graph
    / Stein, Lindeberg–Feller, or martingale-under-adaptive-filtration CLT; OR super-population
    network / spatial sampling — ψ-dependence / weak-dependence / mixing CLT paired with a
    network-HAC / cluster-robust variance) — never an assumed CLT; an optimal / efficient design
    paired with an optimality certificate (KKT / matching variance lower bound / saddle /
    observable approximation-ratio); or exact finite-sample coverage by symmetry / permutation /
    conformal inversion. Assuming the limit law, or asserting a design is optimal without a
    certificate, does NOT qualify.

=== SOFT TIER-MOVERS (apply AFTER the axis check; combine with the hard caps) ===

- **Iff bump.** Headline of the form "θ point-identified / sharp-bounded / equivalent to E'
  iff condition C" (C published-named or a stated new structural condition) tier-bumps one
  notch over a sufficient-only counterpart of equal novelty, when any axis (a)–(j) also
  fires. Necessary-only or sufficient-only WITHOUT a tightness proof does not qualify.
- **Joint-identification bonus.** One structural object (MTR / MTE / response surface /
  nuisance class) point- or sharp-identifying a *family* of named estimands (ATE + ATT +
  ATU + PRTE, or the full MTR/MTE curve) tier-bumps one notch. Exemplars: Mogstad-Santos-
  Torgovitsky (Ecta'18); Heckman-Vytlacil (Ecta'05).
- **Comparator-chain floor (`N-no-comparator-chain`).** Axes (b)/(c)/(e) with only a single
  named comparator and no generic-class promotion → cap at `field`. A flagship statement
  names ≥2 published comparators with a frontier, or promotes to a generic class.
- **Named-focal-object floor (`N-no-named-focal-object`).** A flagship-aspirant statement
  that does not single out a *named* focal mathematical object as the contribution → cap at
  `field`. The object can be any type — operational scalar (critical-δ, threshold T(n,p),
  exposure-weighted score, closed-form ratio), representation (extremal coupling, random-set
  kernel, support-function form, copula identity, bridge-function class), characterization
  (function-class membership, graphical / measure-theoretic condition), algorithm (with
  stated complexity), or new structural object. Math-shape check, NOT empirical validation.
- **Form-explicitness credit (PartialID / bound results).** For the same identified set, a
  closed-form endpoint ranks above an LP / inf-sup framing: reducing a published LP or
  inf-sup bound to closed form is a creditable advance under axis (b)/(e) even when the set
  is unchanged. Credit only, NOT a floor — never cap a result merely for being an optimization.
- **Estimation-duality floor (`N-no-estimator-duality`, Panel only, conditional).** Panel-
  cluster flagship pairing identification with an estimator (closed-form / M-estimator /
  named optimization / DR-orthogonal moment) + a rate / asymptotic-normality conjecture is
  the typical shape, but NOT required when the identification itself satisfies axis (a),
  (c), (f), (g), (h), or (i). Identification-only Panel WITHOUT such an axis → cap at
  `field`. Exempt for ExactID / PartialID.
- **Frontier-advance floor (`N-no-stat-frontier-advance`, Stat only).** A Stat-cluster
  flagship-aspirant headline must ADVANCE THE FRONTIER in one of: (a) a convergence rate
  paired with a minimax LOWER bound proving it unimprovable (named construction — two-point
  / fuzzy / Le Cam, Fano / M-ary, Assouad / d-cube); (b) a SHARPER bound than a named
  published result (strictly faster rate or smaller constant — state the comparison); (c)
  the FIRST nontrivial bound for a named method/estimand that lacked one; (d) a
  semiparametric efficiency bound paired with an ATTAINING estimator; (e) a coverage
  theorem — either paired with its matching limit law / studentization route, or exact
  finite-sample coverage via symmetry / permutation / conformal inversion (cf. axis (j) /
  M17); or (f) a NONTRIVIAL
  ESTIMATOR — consistency + a convergence rate or a limit law, DERIVED from primitive
  conditions, NOT necessarily sharp or minimax-matched — for a NOVEL causal estimand the
  literature had not previously estimated (the estimand must be causal and carry a stated
  consumer, not a contrived relabeling). A matched pair (a) proves tightness and is the
  strongest form; (b)/(c)/(f) are genuine frontier advances and are creditable at flagship
  when the improvement / first-result / new-estimand is significant. A matching minimax
  LOWER bound is NEVER required — a sharper upper rate ((b)), a first bound ((c)), or a
  novel-estimand estimator ((f)) each advance the frontier on their own. Cap at `field`
  ONLY when the result is a lone bound that is NONE of (a)–(f) — not tight, not a strict
  improvement, not a first, not a novel-estimand estimator. The governing structural exponent (smoothness / overlap-decay /
  sparsity / mixing) must be a named §7 quantity the rate is *derived from*, not a free
  tuning parameter (cross-check the `C-free-epsilon` screen). Exempt for Panel / ExactID /
  PartialID / Experimentation (Panel/ExactID/PartialID: estimation is the M8 rung, not the
  kernel; Experimentation is governed by axis (j) + the CLT-derivation floor below, NOT this
  frontier — an M13 optimal-design result carries no rate and must not be capped here).
- **CLT-derivation floor (`N-no-clt-derivation`, Experimentation INFERENCE seeds — M12 / M14 /
  M15).** An Experimentation inference headline that ASSUMES its limit law / CLT rather than
  DERIVING it from named §7 design / dependency / mixing primitives (or asserts Wald-coverage
  with neither an oracle-design-variance route nor a labeled variance-estimator hypothesis) →
  cap at `field`. An OPTIMAL-DESIGN (M13) seed carries no CLT and is governed by the exhibited-
  object floor instead. Exempt for Panel / ExactID / PartialID / Stat.
- **Identification-novelty floor (`N-id-novelty`, SCM identification lane only).** An
  scm-cluster IDENTIFICATION headline (a new identifying functional for a graph-native
  causal query — M19) must be NEW: a new assumption set (including a genuinely novel
  assumption class), a new query, or a new graph / data configuration that yields
  identification — NOT a restatement of a known ID result (classical backdoor / frontdoor /
  do-calculus ID, or an already-published functional). COMPLETENESS is NOT required (most ID
  papers lack it; requiring it is infeasible) but is PREFERRED and scores higher when the
  assumption is shown NECESSARY or the algorithm COMPLETE for the query class. Cap at `field`
  ONLY when the functional restates a known ID result with no new assumption / query /
  configuration. Exempt for Panel / ExactID / PartialID / Stat / Experimentation.
- **Partial-ID practicality floor (`N-pid-practical`, SCM partial-identification lane only).**
  An scm-cluster PARTIAL-ID headline (M20) must deliver a VALID bound (soundness proved over
  the compatible-SCM class), PRACTICAL (a computable / estimable functional of P_obs), and
  NON-TRIVIAL (strictly inside the logical bounds). SHARPNESS is NOT required — a valid
  relaxed bound stands — but is PREFERRED and scores higher when backed by compatible SCMs
  attaining each endpoint. Cap at `field` ONLY when the bound is trivial (equals the logical
  bounds) or non-constructive (no method to compute it). Exempt for Panel / ExactID /
  PartialID / Stat / Experimentation.
- **Concrete-witness floor (`N-no-concrete-witness`).** Flagship-aspirant Conjectures over a
  generic class without a concrete §9 witness (non-corner, non-Gaussian-limit) → cap at
  `field`.
- **Exhibited-object floor (`N-promissory-object`).** A flagship-aspirant headline naming a
  *constructive* focal object (frontier, threshold, sharp set, witness, bridge algorithm,
  certificate, comparator-divergence DGP) WITHOUT a §9 Exhibit computing that object on a
  worked small instance from §6 primitives → cap at `field` (REJECT-eligible at Stage -0.5
  via the EXHIBIT RE-VERIFICATION PROTOCOL). Distinct from the concrete-witness floor, which
  fires only on "generic class" wording.
- **Tautology screen (`C-tautological-iff`).** Flagship-aspirant `iff` where one direction
  unfolds the definition of an object on the other side → cap at `field`.
- **Definitional-unfold screen (`C-definitional-unfold`, non-iff form).** Flagship-aspirant
  headline whose §10 reduction chain is a pure restatement — §6 definition unfolds + §7
  assumption substitutions + routine machinery (LP duality, FWL, support-function transform,
  do-calculus rule k) — with NO genuinely-new step → cap at `field`. Catches Manski-class
  outer-containment, LP-duality re-application, row-stacking identity, support-function /
  projection unfold. **A single genuinely-new key step CLEARS this screen even if the total
  substantive-step count is < 3** (a new construction, a non-obvious reduction, or a lemma
  whose OWN proof is novel) — short, deep proofs are not penalized; the < 3 count is a
  heuristic for "restatement", never a hard quota against elegant flagship proofs.
- **Free-parameter screen (`C-free-epsilon`).** Flagship-aspirant "ε-gap" / "sharp threshold
  τ" claim with the tuning parameter unconstrained above by any §7-named structural constant
  → cap at `field`. Tie the parameter to a named constant (curvature, identification rank,
  SNR, cumulant gap, mixing coefficient, regularizer scale).

When a tier-bump (iff / joint-ID) and a floor (chain / named-focal-object / duality) both
apply, apply the FLOOR FIRST, then check whether the bump still triggers. A single-comparator
iff-result without generic-class promotion stays at `field`. A two-comparator iff-result
without a named focal object stays at `field`. A two-comparator iff-result WITH a named focal
object promotes to `flagship`.

=== HARD CAPS — negative-result kernels (counterexample / impossibility / sharpness-failure) ===

- `N-strawman` fires (no named published target uses the refuted method) → at most `letter`
  (Stage -0.5) / `incremental` (Stage 0.5).
- `N-witness-trivial` fires (target named but witness degenerate / single hand-tuned instance)
  → at most `field` (Stage -0.5) / `subfield` (Stage 0.5).
- `flagship` requires BOTH a named published target AND a generic-class obstruction (a generic
  region of observable laws / proxy systems / graphs — non-degenerate, not a single instance).

=== END FLAGSHIP RUBRIC ===
