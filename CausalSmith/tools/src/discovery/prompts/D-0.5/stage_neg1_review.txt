=== HEADLINE PLACEMENT + SCOPE-CAVEAT ADJUDICATION (closes the -0.5 → 0.5 relitigation leak) ===

- **Headline placement.** The proposal's strongest / most computable result must be a `Conjecture` / `Open-ended Question` (a §8 solve target → it headlines §9), NOT a supporting `Theorem` (those land in the §11 prelude). - **Scope-caveat adjudication.** For every honest-scope flag in §11 ("assumed not proved", "not a construction", "downgraded to a premise"), DECIDE now — do not leave it for Stage 0.5 to relitigate: either (a) ACCEPT it as a conditional claim — bind the headline to the conditional form, cap the tier per the derived-vs-assumed gate, and list it in `accepted_scope_caveats` (Stage 0.5 then treats it as settled, not a fresh correctness defect); or (b) REQUIRE RESOLUTION via `REVISE` (drafter proves it or restates the headline to not claim it). Pick (b) only when the caveat undercuts the headline contribution at the claimed tier.
  - **Caveat-eligibility guard (a caveat may narrow a regime, NOT assume the contribution).** Option (a) is FORBIDDEN — you MUST pick (b) REVISE — when the caveated assumption is load-bearing in either way: (i) it ASSUMES a **named comparator's defining estimand** (its weights / loading / influence function / score / estimand formula) instead of DERIVING it from that comparator's published definition (regression normal equations, the estimator's own formula); or (ii) it ASSUMES the **very object the headline characterizes** (the frontier / bound / threshold / equivalence condition), so the theorem reduces to a property of the assumed object; or (iii) it STRENGTHENS a standard assumption beyond its literature form to enable the target rate — a sup-norm where L2 is the standard nuisance condition, a known nuisance where estimating it is the point, a clip / trim / identification condition only an oracle satisfies — so the rate/efficiency is established only for a stronger-than-standard model, not the stated class. These are comparator-fidelity / tautology / rate-buying defects, not regime scope: blessing them lets the proposal "assume the comparator's weights, then prove a property of the assumed weights" (a comparator loading assumed in §7 rather than derived from its published spec, so the iff is near-immediate and the comparator is a stand-in, not the published estimator). A legitimate scope caveat restricts the regime / class / convention ("sharp under saturation", "for bounded-overlap laws"); it may NOT supply a named comparator's estimand or the characterized object. Classify (i) as `novelty` (comparator not faithfully represented — the named estimator's estimand must be derived from its spec), (ii) as `correctness` (tautological / definitional unfold), and (iii) as `novelty` (the target rate is not established for the standard class — require the rate achievable under standard assumptions, slower if need be, or an OEQ on whether the target is attainable under them; a better rate must come from a better estimator/proof, not a stronger hypothesis). **Exception to (iii):** a new non-standard assumption that is the proposal's STATED contribution, justified as realistic and satisfied in real-world settings (a concrete class or example), is a legitimate new-condition paper — do NOT flag it; (iii) fires only on an instrumental strengthening with no independent real-world justification, present only to reach the rate. This guard is tier-independent (it fires at `field`, not only flagship, since it does not rely on the flagship-gated SC10/SC11 screens). Require the drafter to DERIVE the comparator estimand from its published spec, or restate the headline so it no longer claims the comparison / the assumed object. ALSO record in `accepted_scope_caveats` any **tier cap you apply because the flagship focal object is itself §7-assumed or a definitional unfold** (the derived-vs-assumed gate) — set `caveat` to the assumed object and `bound_claim` to the capped tier, so Stage 0.5 does not re-apply the same cap or re-flag the same assumed object as a new `C-definitional-unfold`/correctness defect.

A flagship-aspirant scm partial-ID headline delivers a bound that is TRIVIAL (equals the logical bounds) or NON-CONSTRUCTIVE (no computable / estimable functional of P_obs), or whose SOUNDNESS over the compatible-SCM class is not established. Soft flag: caps `publishability_tier` at `field`. Sharpness is NOT required (PREFERRED, not gated): a valid, practical, non-trivial relaxed bound clears this code; a sharp bound backed by compatible SCMs attaining each endpoint scores higher but is never required. Exempt for Panel / ExactID / PartialID / Stat / Experimentation. Flag with the offending §8 label.
`N-no-concrete-witness` — A flagship-aspirant `Conjecture` (or `Theorem`) phrased over a *generic open class* — "for generic P in O", "on an open subset of admissible laws", "for a residual set of parameters", "for almost every G in an open family" — does NOT exhibit a single *concrete* instance (a specific parametric family beyond the corner reduction: Gaussian + ε-perturbation, finite-support distribution with named atoms, planar SVAR with a specific rotation angle, a named DAG with concrete edge counts) for which §10's proof sketch has an explicit route. The Gaussian limit / trivial reduction in §9 does NOT qualify — that is the corner where the claim degenerates, not a witness of the generic kernel. Distinct from `N-witness-trivial` (which targets *counterexample* kernels with degenerate observable laws); this code targets *positive* flagship kernels that promise generic-class scope without exhibiting a non-degenerate instance. Hard cap: flagship-aspirant generic-class kernels without a concrete §9 witness fire this flag and cap at `field`. Remedy: either (i) name the concrete witness in §9 and ensure §10 routes through it, or (ii) downgrade the §8 statement from "generic open class" to the witness class directly.
`N-promissory-object` — A flagship-aspirant `Theorem` / `Conjecture` whose headline names a **constructive focal object** — a frontier T(P), threshold τ*, sharp set S*, witness P*, bridge-function class H, identification algorithm, certificate, comparator-divergence DGP, or any object the kernel claims to produce rather than assume — does NOT include a §9 exhibit block computing that object on a worked small instance from §6 primitives. Distinct from `N-no-concrete-witness` (which fires only when the headline is phrased over a "generic open class"); this code fires on **any** flagship-aspirant headline naming a constructive focal object, regardless of class-wording. The diagnostic: execute the EXHIBIT RE-VERIFICATION PROTOCOL above. The flag fires when ANY of the following hold: (i) no §9 Exhibit block exists for the focal object; (ii) the proposer's `SC10_focal_object_exhibited` JSON field is a bare one-liner with no structured transcript (defining equation, RHS-symbol provenance, substituted equation, computed value, invariant checks, assumption-substitution screen); (iii) any RHS symbol load-bearing for the flagship novelty is sourced from §7 assumed quantities — the focal object is assumed, not derived; (iv) any invariant check in the transcript FAILS upon re-verification; (v) the proposer's computed value disagrees with your independent recomputation; (vi) for non-equivalence claims, the exhibit fails to produce distinct values for the named estimators, or the difference is a normalization artifact. **Verdict routing:** this flag is REJECT-eligible at flagship under triggers (i)–(v) — the headline construction is structurally unsound and the orchestrator's revise budget should not be spent polishing it. Trigger (vi) and ambiguous cases route to REVISE with field-cap; the soft cap to field remains as the secondary fallback. Remedy: drafter adds a §9 Exhibit computing the focal object on a small instance (T=3 panel, 2-arm treatment, finite support, named small DAG) from §6 primitives with an internal-consistency check AND pastes the full transcript into `SC10_focal_object_exhibited`, OR downgrades the headline to acknowledge the object is assumed rather than produced.

--- C — SOUNDNESS (well-posedness + coherence + scope, NOT verification, NOT Lean-feasibility) ---

`C-wellposed`     — Missing residual term, undefined object, or terminology slip (mean-reverting vs anti-persistent vs persistent, switch-on vs switch-off, never-treated vs not-yet-treated, etc.). ALSO fires when the **target estimand is not a well-defined functional of the observed-data law**: an *at-a-point* estimand — a conditional mean / regression / density / quantile evaluated at a fixed covariate value (`E[·∣X=x_0]`, `τ(x_0)`, `f(x_0)`) — is defined only up to a `P_X`-null set, so its value at `x_0` is NOT a functional of `P` unless §7 states an explicit regularity/version condition (continuity of the target at `x_0`, Lebesgue point, density positive + continuous near `x_0`). A smoothing/localization bound that merely caps the point value (e.g. `|θ_h(P)−τ(x_0)|≤b_n`) *presupposes* the point value, it does not well-define it — do not let the localization machinery (which IS well-posed) mask an ill-posed target. **Topic-level tension (load-bearing):** when the proposal's selling point is *structure-agnostic / black-box / assumption-lean* (no smoothness), an at-a-point target is near self-contradictory — the regularity that would well-define `τ(x_0)` is exactly the smoothness the framing disclaims, and adding it tends to collapse the contribution into the smooth-model prior art (e.g. Hölder pointwise minimax). This is a topic defect → REVISE/pivot, not a missing-symbol slip. Remedy: restate the estimand as the smoothed functional `θ_h(P)` (a genuine law functional — then re-check it is not already covered by prior art), or pivot off the at-a-point target.
`C-sanity`        — A claimed §9 corollary / reduction does not actually reduce under the stated assumptions (e.g. "reduces to existing T_k at T=2" but the algebra fails).
`C-coherence`     — Hypotheses cited in `Theorem N` / `Conjecture N` are missing from §7 Assumptions; OR the "what is new" line in §4 does not match the kernel claim in §8; OR notation clashes between sections.
`C-proof-sketch`  — §10 "Proof-sketch route" is missing, hand-waves the reduction, or names no concrete classical theorem / algebraic identity / boundary case the proof would reduce to. (NOTE: do NOT flag for missing Mathlib lemmas or Causalean Lean-file references — Stage 1 owns the formalization-feasibility gate; §10 is a math sketch, not a Lean inventory.)
`C-scope`         — §11 Honest scope flags is empty despite at least one `Conjecture` present in §8 (self-deception).
`C-tautological-iff` — A flagship-aspirant `Theorem` / `Conjecture` is stated as `A iff B` (or `A iff C ≥ τ*` / `A iff P ∈ S*`) where ONE direction unfolds the very definition of an object on the other side. The diagnostic test: take the §6 or §7 definition of the threshold / set / index on the RHS — if it reads "τ* := inf {c : A holds}" or "S* := {P : A(P) holds}" or any other "smallest / first / critical / minimal value at which A holds" form, then "A iff C ≥ τ*" / "A iff P ∈ S*" is a definitional unfold, not a frontier. The math is correct but vacuous as a flagship contribution because the iff carries no information beyond the definition. Distinct from `C-sanity` (which targets *failed* reductions) — this code targets *trivial* reductions disguised as frontiers. Hard cap: this flag caps `publishability_tier` at `field`. Remedy: either (i) state the threshold via an *independent* characterization (closed-form in observables, named in published comparator) so the iff genuinely transports information, or (ii) drop the iff framing and state the inclusion / monotonicity directly.
`C-free-epsilon`  — A flagship-aspirant claim of the form "ε-gap exists" / "sharp threshold at τ" / "non-trivial margin holds" has the tuning parameter (ε, τ, δ, margin) that is *defined* in §6 but is NOT bounded above by any other named §7 quantity (curvature constant, identification rank, signal-to-noise ratio, cumulant gap, mixing coefficient, regularizer). When the parameter is unconstrained above, the claim is trivially achievable by taking the parameter sufficiently small (for "ε exists" type) or sufficiently large (for "τ-threshold" type), and the substantive content is hidden. Distinct from `C-wellposed` (which targets undefined objects) — this code targets *defined but unconstrained* tuning parameters that trivialize the claim. Soft flag: caps `publishability_tier` at `field`. Remedy: tie the tuning parameter to a published or §7-named structural constant (e.g. "ε ≤ c_w(P)" with c_w(P) the curvature in Assumption X, "τ ≥ κ_min where κ_min is the smallest non-zero cumulant difference") so the claim is non-trivially constrained.
`C-definitional-unfold` — A flagship-aspirant `Theorem` / `Conjecture` whose §10 proof-sketch route is a *pure restatement*: it reduces structurally to §6 definition unfolds + §7 assumption substitutions + routine machinery, with NO genuinely-new step. Distinct from `C-tautological-iff` (iff-form specifically); this catches the broader Manski-class outer-containment / LP-duality re-application / support-function unfold / row-stacking / projection-identity shapes. **Diagnostic:** write §10's chain as `headline ← [unfold §6 defs] ← [apply §7 assumption] ← [step 1] ← ... ← [classical result]`; reflexive applications of standard machinery (LP duality, FWL, support-function transform, do-calculus rule 2) count as ZERO. Fire the flag when the chain contains NO genuinely-new key step. **A single genuinely-new key step — a new construction, a non-obvious reduction, or a lemma whose OWN proof is novel — CLEARS this flag even if the total count of substantive steps is < 3.** Short, deep proofs are not penalized; `k < 3` is a heuristic for "restatement", not a hard quota. Hard cap when it fires: `field`. Remedy: either (i) identify / add the genuinely-new step and explain why a top referee finds it non-obvious, or (ii) downgrade the headline to a corollary / specialization.

NOT in scope (do not flag): "the conjecture is unproven", "the conjecture is hard", "I doubt this can be proved". See the scope guard above.

- `flagship` (J. Econometrics / QE / ET / Annals)
- `field` (good field journal)
- `letter` (Economics Letters / short note)
