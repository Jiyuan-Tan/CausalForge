=== PROPOSAL-LEVEL SCOPE GUARD (LOAD-BEARING) ===

Statements are `conjecture`/`theorem`/`proposition`, all `to-prove` — intentional open kernels. Do NOT mark a conjecture as a soundness failure for lack of proof. Soundness here covers ONLY: (i) is the statement mathematically well-posed (no missing residual terms, no undefined objects, no terminology errors); (ii) does it reduce to its claimed sanity reductions under standard special cases; (iii) are the named prior-work comparisons accurate. Whether an open conjecture is TRUE is out of scope.

**Conjecture vs open-ended question.** A `conjecture` COMMITS a candidate construction (the relevant `definition.construction` is a concrete expression); an open-ended question gives only a Handle (a named technique + target object) in `definition.construction`. Treat both as first-class open content. Two adjustments:
- If a `conjecture` commits an object (bound / threshold / closed-form / estimator / coupling / sign / rate / equivalence) whose value is asserted by ANALOGY / heuristic to a known case rather than constructed by the proposal's own route, recommend relabeling it DOWN to an open-ended question (the construction route becomes the Handle), so D0 gets a constructive mandate rather than a verify-this-guess mandate. Score as if correctly labeled. (A minimax lower bound asserted to "match the ATE rate" is the common instance; the rule is general.)
- A bare Handle ("design an estimator for τ*") with no concrete starting move fires `N-no-named-focal-object`. Recommend sharpening the Handle, demoting to a supplementary remark, or pivoting.

=== SCOPE-CAVEAT ADJUDICATION (closes the -0.5 → 0.5 relitigation leak) ===

For every honest-scope caveat (an assumption flagged "assumed not proved", a premise the kernel leans on), DECIDE now: either (a) ACCEPT it as a conditional claim — bind the headline to the conditional form, cap the tier per the derived-vs-assumed gate, list it in `accepted_scope_caveats` (Stage 0.5 treats it as settled); or (b) REQUIRE RESOLUTION via REVISE.

**Caveat-eligibility guard (a caveat may narrow a regime, NOT assume the contribution).** You MUST pick (b) REVISE when a load-bearing assumption: (i) ASSUMES a named comparator's defining estimand (weights / loading / influence function / score / formula) instead of it being DERIVABLE from that comparator's published spec → classify `novelty` (comparator is a stand-in); (ii) ASSUMES the very object the headline characterizes (the frontier / bound / threshold / equivalence condition), so the statement reduces to a property of the assumed object → classify `correctness` (tautological / definitional unfold); (iii) STRENGTHENS a standard assumption beyond its literature form to enable the target rate (a sup-norm where L2 is standard, a known nuisance where estimating it is the point, an oracle clip/trim) → classify `novelty` (require the rate under standard assumptions, slower if need be, or an OEQ on attainability). **Exception to (iii):** a new non-standard assumption that IS the proposal's stated contribution, justified as realistic and satisfied in real settings, is a legitimate new-condition paper — do not flag; (iii) fires only on an instrumental strengthening with no independent real-world justification. This guard is tier-independent (fires at field too). Record any tier cap you apply (because the flagship focal object is itself an assumed quantity or a definitional unfold) in `accepted_scope_caveats` so Stage 0.5 does not re-flag it.

=== CHECKS (three dimensions — S / N / C) ===

Each finding gets exactly one code. Routing: S → structural patch; N → novelty redraft/pivot; C → statement-level redraft. (Most structural well-formedness is already gated, so S findings are rare here.)

--- S — STRUCTURE ---
`S-struct`  — the kernel's strongest / most computable result is NOT a `conjecture`/OEQ but a supporting `theorem` (reclassify the headline as a conjecture); OR the kernel sits in infrastructure (SCM/PO/Graph) rather than an ExactID/PartialID/Panel/Stat result (substrate misuse).
`S-cite-thin` — `related_work` cites fewer than 3 published papers, or cites without a relevance sentence each.

--- N — NOVELTY (per-statement, two axes: repo + published; `overall` is `new` only if both are `new`) ---
`N-repo` — repo axis already-known (kernel duplicates an in-catalogue proposal/note). Cite by qid + headline.
`N-pub` — published axis already-known (covered by a named paper). Cite bibkey.
`N-mischar` — a named prior-art comparison (in `related_work` or a statement's `gap`) misstates the cited paper's claim. Cite paper + the misstatement.
`N-thin-survey` — fewer than 3 plausible prior-art candidates for a flagship statement, or you found a closer published result the drafter missed. Name it.
`N-all-theorem-trap` — every statement is a `theorem` with no `conjecture` and no OEQ (hidden open content). Do not fire if there is ≥1 conjecture or ≥1 well-formed OEQ.
`N-strawman` — a counterexample / impossibility kernel names no specific published paper/estimator/workflow that uses the refuted method (bibkey + quoted claim required).
`N-witness-trivial` — a counterexample's witness sits on a degenerate law (Y constant a.s., arm collapsed, variation hidden in counterfactuals). Ask for a non-degenerate witness or a generic-class obstruction.
`N-no-implication` — a statement's `consumer` is generic ("future work", "broad interest", "of independent interest") with no named paper / bibkey / applied domain / open problem. Cite the statement + quote the phrasing.
`N-comparator-drift` — the prose names a published comparator ("extends/sharpens X") but NO statement is phrased as a strict tightening / iff frontier / equivalence / non-representability against that comparator's published result. Remedy: add such a statement, downgrade "extends" to "informed by", or drop the comparator.
`N-thin-anchor` — a flagship-aspirant statement cites a paper by bibkey alone with no location anchor (section / equation / page / quoted open problem) tying the seed to the specific claim. Field-tier exempt.
`N-no-named-focal-object` — a flagship-aspirant statement names no single focal object as the contribution (only an inf/sup over an unspecified class, a set-of-laws statement, or a qualitative existence claim). Soft cap to field unless another flagship axis holds.
`N-no-comparator-chain` — a flagship-aspirant statement names only ONE comparator and does not promote to a generic-class statement. Soft cap to field unless the single comparator is paired with a generic-class obstruction.
`N-no-estimator-duality` — Panel flagship: identification not paired with an explicit estimator OR an asymptotic (rate/normality/efficiency) conjecture. Soft cap to field UNLESS the identification itself opens a new regime / wider class / new target / new structural object / framework bridge / constructive algorithm (axes a/c/f/g/h/i). Exempt for ExactID/PartialID.
`N-no-stat-frontier-advance` — Stat flagship: the result advances the frontier in NONE of the recognized ways — it is a lone bound that is neither (a) matched by a minimax lower bound (named two-point / Le Cam / Fano / Assouad), nor (b) a strict improvement over a named published bound (faster rate / smaller constant), nor (c) the first nontrivial bound for a named method/estimand, nor (d) an efficiency bound with an attaining estimator, nor (e) a coverage theorem with a matching limit law — OR the governing structural exponent is a free tuning parameter rather than a named quantity the rate is derived from. Soft cap to field. A matched pair PROVES tightness (strongest), but a SHARPER bound or a FIRST nontrivial bound is itself creditable and does NOT fire this code. Exempt for Panel/ExactID/PartialID.
`N-id-novelty` — SCM identification lane (M19): the identifying functional RESTATES a known ID result (classical backdoor / frontdoor / do-calculus ID or an already-published functional) with no new assumption, query, or graph / data configuration. Soft cap to field. Completeness NOT required (preferred): a new functional under a stated (possibly novel) assumption clears it. Exempt for Panel/ExactID/PartialID/Stat/Experimentation.
`N-pid-practical` — SCM partial-ID lane (M20): the bound is TRIVIAL (equals the logical bounds) or NON-CONSTRUCTIVE (no computable/estimable functional), or its soundness over the compatible-SCM class is not established. Soft cap to field. Sharpness NOT required (preferred): a valid, practical, non-trivial relaxed bound clears it. Exempt for Panel/ExactID/PartialID/Stat/Experimentation.
`N-no-concrete-witness` — a flagship-aspirant statement phrased over a generic open class ("for generic P in O", "almost every G") that commits NO construction at all (no witness family, no route). Caps at field. Remedy: commit a construction, name the route, or narrow the class. Does NOT fire when the statement — or a definition it depends on — already commits an explicit construction / witness family whose class-membership / non-degeneracy is a stated `to-prove` obligation: instantiating a concrete non-degenerate witness and verifying it is the SOLVER's (D0) job, not the proposal's. (This keeps the code from contradicting N-promissory-object, which only asks that a construction be committed — not that it be instantiated.)
`N-promissory-object` — a flagship-aspirant CONJECTURE names a constructive focal object (frontier, threshold, sharp set, witness, bridge class, algorithm) but does NOT commit a real construction: the relevant `definition.construction` is empty, or is a by-analogy guess, or assumes (rather than builds) the object from primitives. An EXISTENTIAL placeholder ("for any set C_n there exist amplitudes δ achieving the separation", "∃ a least-favorable family with …") does NOT count as committing a construction — it is itself promissory and re-fires this code. Remedy (PREFER THE LIGHTWEIGHT ONE): name the construction ROUTE as a Handle and relabel the claim as an open-ended question (the full witness is the proof's job) — OR, only if it is genuinely short, commit an EXPLICIT well-posed construction (named perturbations / witnesses with stated bounded pairwise divergence, not an ∃-placeholder). For a minimax converse specifically, prefer route+relabel over attempting the witness inline. REJECT-eligible at flagship when the headline construction is structurally absent; otherwise REVISE with field-cap.

--- C — SOUNDNESS (well-posedness + coherence + scope; NOT verification, NOT Lean) ---
`C-wellposed` — missing residual term, undefined object, terminology slip; OR the `target_estimand` is not a well-defined functional of the observed-data law (an at-a-point estimand `E[·|X=x0]` / `τ(x0)` with no stated regularity/version condition is defined only up to a null set — and if the selling point is structure-agnostic, the regularity that would well-define it is the very smoothness disclaimed → topic defect, REVISE/pivot).
`C-sanity` — a claimed sanity reduction does not actually reduce under the stated special case (the algebra fails).
`C-coherence` — a hypothesis cited by a statement is missing from `assumptions`; OR the prose novelty claim does not match what the statements claim; OR notation clashes.
`C-scope` — the proposal leans on an open conjecture but flags no honest scope at all (self-deception) — only when a real open risk is unacknowledged (presence of the optional `honest_scope` field is not itself required by this code; the gate handles required prose).

**ASSUMPTION AUDIT (per assumption; the line-21 caveat guard handles laundering — these grade the assumption set on its own terms).** Walk EVERY entry of `assumptions`:
`C-assumption-nonstandard` — an assumption tagged `standard:{name,cite}` is not in fact the standard condition under that name (misnamed, non-standard as stated, or the cite does not support / does not exist for it). Verify each `standard` tag against the literature. Remedy: correct the name/cite, or retag `novel` (then it needs a `justification`).
`C-assumption-impractical` — a `novel` or non-standard load-bearing assumption is not plausibly satisfied in realistic settings AND its `justification` does not credibly establish applicability (empty / circular / hand-wavy justification for a strong condition). Distinct from the (iii) laundering guard: that fires on instrumental strengthening to enable the target rate; this fires whenever a load-bearing condition's real-world validity is simply unestablished.
`C-assumption-redundant` — the assumption set is non-minimal or internally inconsistent: one assumption implies/subsumes another, two are jointly unsatisfiable, or a statement leans on a hypothesis strictly stronger than it needs. Remedy: drop/weaken the redundant condition, reconcile the conflict, or split.
`C-assumption-nonprimitive` — a condition that carves a class (a `by_member_properties` entry, or a load-bearing membership assumption) is stated as a property of a NAMED ESTIMATOR / procedure rather than of the LAW and its primitive functionals (the propensity, the outcome regressions, P). The tell: a membership assumption whose condition constrains a specific estimator, e.g. `‖ĝ − g‖ = O_P(r)` for a named `ĝ` (a class of laws must not depend on a procedure). Remedy: restate as a property of the nuisance — `g is L2(P_X)-estimable at rate O_P(r)` (existential over estimators) — and pass the estimator as a theorem/construction INPUT, not a membership condition. (Distinct from C-assumption-impractical, which grades real-world validity; this grades law-vs-procedure category.)

**VACUITY defects (C-tautological-iff, C-free-epsilon) — fire at ANY tier.** These are NOT flagship floors: a content-free result is unpublishable at *every* tier, so apply them to any statement, field or flagship. When they fire, REVISE (the kernel must carry real content); at flagship they ALSO cap the tier, but the defect itself is tier-independent.

`C-tautological-iff` — a statement `A iff B` where one direction merely unfolds the definition of an object on the other side (the threshold/set is *defined* as "the value/set at which A holds", so the iff transports no information). Vacuous at any tier. Remedy: state the threshold/set via an INDEPENDENT characterization (closed form in observables, a named published comparator) so the iff carries information, or drop the iff and state the inclusion/monotonicity directly.
`C-free-epsilon` — an "ε-gap exists / sharp threshold at τ / non-trivial margin holds" claim whose tuning parameter (ε, τ, δ, margin) is *defined* but NOT bounded by any named structural quantity, so the claim is trivially achievable by taking the parameter small/large enough. Vacuous at any tier. Remedy: tie the parameter to a named structural constant (curvature, identification rank, cumulant gap, …) so the claim is non-trivially constrained.
`C-definitional-unfold` — a statement whose claim reduces to definition unfolds + assumption substitutions with too little substantive content (outer-containment / LP-duality re-application / support-function unfold / projection identity). TWO regimes: (i) if it reduces to ZERO substantive steps (a pure definitional restatement), it is a VACUITY defect → fires at ANY tier, REVISE; (ii) if it carries 1–2 substantive steps but fewer than 3, that is a FLAGSHIP floor → caps a flagship-aspirant headline at field (a field result may legitimately be a clean 1–2-step reduction). Judge from the statement + its dependencies (no proof route exists at this stage — assess derivability structurally). Remedy: carry more substantive steps, or downgrade the tier framing.

NOT in scope (do not flag): "the conjecture is unproven / hard / I doubt it". See the scope guard.

=== PUBLISHABILITY TIER ===

Assuming the conjectures resolve favourably, rate `flagship` / `field` / `letter` / `not-publishable`. A proposal is flagship-eligible iff the eventual derivation would satisfy at least one regime axis (a)–(j) in the FLAGSHIP RUBRIC. Apply the derived-vs-assumed gate now: if the flagship novelty rides on an assumed quantity (a `standard`/`novel` assumption that supplies the characterized object), cap at field NOW.

Two distinct mechanisms — do not conflate:
- **VACUITY (any tier).** A tautological iff, a trivially-achievable free parameter, or a pure definitional restatement is content-free and unpublishable at EVERY tier → rate `not-publishable` (or REVISE for a restatement that could carry content) regardless of flagship eligibility. These are not "merely not flagship".
- **FLAGSHIP FLOORS (cap to field).** comparator-chain, named-focal-object, estimation-duality, stat-frontier-advance, concrete-witness (and the 1–2-substantive-step definitional-unfold) cap a flagship-aspirant headline at field but do NOT disqualify a field paper. Apply with floor-before-bump ordering and the negative-result hard caps.

When a floor OR a vacuity defect fires, ALSO emit its N-/C- code in `novelty_flags`/`soundness_flags` (the orchestrator's tier-saturation auto-promote depends on those arrays being empty).

=== ORCHESTRATOR-LEVEL TIER-SATURATION RULE (informational) ===

The orchestrator auto-promotes a REVISE→ACCEPT when the SAME angle gets `verdict=REVISE` at or above the novelty floor with EMPTY `novelty_flags` AND `soundness_flags` for 3 consecutive versions (and early-stops an angle that never reached the floor by v3). So: emit every genuine N-/C- flag (the rule never fires while one is present), but do NOT preemptively soften real flags to trigger it.

=== STRENGTHENING PATHS ===

REQUIRED regardless of verdict (even ACCEPT can be sharpened). ≥2 distinct, actionable suggestions, each tagged low/medium/high-effort (tighten an assumption, sharpen a conjecture, add a boundary corollary, replace a qualitative claim with a rate, name a missed comparator).

Replace every `"TODO: ..."` with content, every `0` placeholder with the real count, and expand each array's `_prototype` into the real items (`[]` if none). Drop every `_`-prefixed key. The S/N/C flag-code enums are documented above. Single JSON object, last non-empty line of stdout.

VERDICT routing:
- `ACCEPT`: novelty kernel genuine and well-posed on both axes; `novelty_flags` / `soundness_flags` empty (or trivial typo only).
- `REVISE`: kernel survives but ≥1 patchable flag fires (S; N-mischar/N-thin-survey; C-wellposed/C-coherence/C-scope; C-assumption-nonstandard/C-assumption-impractical/C-assumption-redundant/C-assumption-nonprimitive; OR a VACUITY defect — C-tautological-iff / C-free-epsilon / pure-restatement C-definitional-unfold — that the drafter can fix by an independent characterization or by constraining the parameter, at ANY tier).

Do NOT REJECT solely because a conjecture is unproven.
