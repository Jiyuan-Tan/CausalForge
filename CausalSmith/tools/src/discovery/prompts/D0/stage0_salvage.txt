=== STAGE 0.7: SALVAGE A REFUTED CONJECTURE ===

A Stage 0 derivation refuted this conjecture with an explicit counterexample, and
Stage 0.5 confirmed the refutation is GENUINE (not a solver error). Your job is to
decide whether the refutation kills the substance or only a fixable corner, and —
when fixable — to turn it into the nearest POSITIVE result via the minimal NATURAL
repair. This is Lakatos lemma-incorporation made honest: you may incorporate the
property the witness exploits as a hypothesis or correct the object/rate, but ONLY
if the repair is a recognized convention, not a patch invented to exclude the
witness. The default is NOT "salvage at any cost" — a forced salvage that guts the
question is worse than an honest refutation.

Read, in order: the shared setup (the formal objects), the refuted verdict JSON
(its headline_statement, witness_construction_if_refuted, refutation_artifact_path),
the refuted fragment §13 (the counterexample), and every confirmed-sibling anchor.

--- STEP 1: CLASSIFY THE REFUTATION ---
Determine which TYPE this is and record classification + classification_reason:

  Type A — OBJECT / HYPOTHESIS BUG. The witness exploits a degenerate corner of
    the estimator/object definition or a missing regularity hypothesis; the CORE
    kernel claim is untouched. Diagnostic: the witness lives in a special regime
    (a fixed/boundary parameter value, a degenerate support, an un-regularized
    construction) and the interesting regime the question is ABOUT is not where
    the witness bites.

  Type B — RATE / CONSTANT CORRECTION. The witness proves the claimed quantity
    (rate, bound, variance, constant) is wrong, but it IMPLIES the correct one.
    Diagnostic: the witness pins a value that contradicts the headline number yet
    is itself a clean target the achievability/converse can be re-balanced to.

  Type C — FATAL. The witness destroys the premise: the estimand is not
    identified, no estimator beats a trivial benchmark, or the kernel mechanism
    is false on the very regime the question is about. Type C is NOT salvageable —
    set salvage_outcome=unsalvageable and NAME the obstruction. Do not downgrade a
    Type C into a vacuous Type A by adding a hypothesis that erases the question.
    Type C is the OBJECT dying, not the conjectured VALUE being wrong: reserve it
    for a dead question (estimand unidentified, no estimator beats a trivial
    benchmark, mechanism false on the regime). If the focal OBJECT is still
    well-defined and only its conjectured value/rate/threshold is refuted — its true
    value remaining a sharp, open target — that is NOT Type C: route to Type B if the
    witness implies the value, else to the RE-TARGET path (Step 2'). A refutation
    redirects the question; it does not kill the topic.

ANCHOR RULE: if a confirmed sibling exists (e.g. a confirmed lower bound paired
with a refuted upper bound), the mend has a TARGET — the salvaged side must meet
the confirmed side's rate/structure. If there is NO confirmed sibling, you have no
anchor; bias strongly toward Type C unless the mend stands entirely on its own.

COUNTEREXAMPLE-AS-ANCHOR: the refuting witness is itself a BOUND on the true object
(an information / Le Cam witness is a sharper lower bound; a constructed failing law
upper-bounds what is attainable). When the witness bound is SHARPER than any
confirmed sibling, promote the witness to the anchor — and to a standalone positive
result — rather than discarding it as a refutation artifact. A confirmed sibling the
witness proves non-tight is NOT a valid mend target.

--- STEP 2: MEND (Type A or B only) ---
Extract the EXACT property the witness exploits (record it as witness_property),
then propose the minimal repair (record mend.kind + mend.description):
  - added-hypothesis: a primitive regularity condition that rules out the witness
    regime (Type A). State it in assumption form.
  - object-correction: fix the estimator/object definition so the witness regime
    is handled correctly without excluding it from the class (Type A).
  - rate-correction: replace the claimed rate/constant with the witness-implied
    one and re-derive achievability/converse to match (Type B).
Prefer object-correction over added-hypothesis when the bug is in YOUR construction
(do not assume away your own mistake). Anchor the mend to any confirmed sibling.

--- STEP 2': RE-TARGET (object alive, value only bracketed) ---
When the object survives but the witness pins only a BOUND, not the exact value (no
matched pair is yet derivable), do NOT mark unsalvageable and do NOT retreat to a
weaker conditional sub-claim that merely scopes to the sub-region where the refuted
value happens to hold (that abandons the ambition). Instead, at the SAME ambition as
the original (still the matched / sharp target): (i) promote the witness bound to a
standalone positive result (refutation-with-positive-result — record it as the
mended/positive claim); (ii) attempt a crude achievability of the opposite side to
bracket the true value; (iii) re-target the object as an OPEN exact-value question
(salvage_outcome=salvaged-partial, with the open exact value recorded as the gap).
The deliverable is a sharper bound + a precise open question, never a silent
downgrade to a lone one-sided or conditional result.
Emit a gate-valid salvaged-partial: classification B, mend.kind=rate-correction (you
correct the conjectured value and add NO hypothesis), mend.justification_cite = the
bibkey of the technique that ESTABLISHES the promoted bound (the Le Cam / two-point /
fuzzy-hypothesis source the witness already uses — this is a real cited convention,
not monster-barring), guards all pass (vacuity: the class is non-empty; substance:
the motivating regime is retained — you did NOT scope it away; naturalness: no
witness-tailored hypothesis was added), mended_headline_statement = the promoted
(provable) bound, and obstruction omitted. The open exact value is the recorded gap.

--- STEP 3: GUARDS (ALL THREE must pass or the salvage FAILS) ---
Record guards.{vacuity,substance,naturalness}, each pass + evidence:
  (a) VACUITY FLOOR — exhibit a concrete member of the mended class (a specific
      law/instance satisfying every hypothesis incl. the new one). If you cannot
      name one, the mended class may be empty → fail.
  (b) SUBSTANCE — the motivating regime of the ORIGINAL question must survive the
      mend. Name the regime the question is about and show the mended class still
      contains it. A repair that only survives by excluding the interesting regime
      is monster-barring → fail (and is really Type C).
  (c) NATURALNESS / ANTI-MONSTER-BAR — the added hypothesis / corrected object
      must be a recognized convention in the literature, NOT tailored to exclude
      this witness. TEST: would a careful author state this hypothesis BEFORE
      seeing the counterexample? Fetch the convention via web_search and cite it in
      mend.justification_cite (bibkey + arXiv-id/DOI). "Exclude the witness's exact
      construction" is NOT natural → fail.
If any guard fails, set salvage_outcome=unsalvageable and NAME the obstruction in
obstruction_if_unsalvageable. Never report a salvage with a failing guard.

--- STEP 4: RE-DERIVE (only if all guards pass) ---
Prove the mended claim from PRIMITIVE conditions (reuse the project's end-to-end
preference: construct the object explicitly, prove its property from i.i.d./moment/
tail primitives; do not assume a dischargeable high-level input). Produce the
mended_headline_statement. Write a lineage_note in the form:
  "<original claim> refuted by <witness>; mended to <C'> via <H>; H justified by <cite>".
salvage_outcome=salvaged-confirmed if the mended claim is fully derived;
salvaged-partial if a genuine gap remains (record it like a partial conjecture).

--- HONEST-VERDICT RULE ---
Do NOT manufacture a salvage by (i) weakening the estimand, (ii) adding a
witness-tailored hypothesis, (iii) hiding a gap, or (iv) re-labelling a Type C as a
narrowed Type A. An honest unsalvageable with a named obstruction is a SUCCESS for
this stage. The orchestrator re-checks all three guards and the mended verdict; a
laundered salvage is rejected.

When a mend ADDS or REPAIRS a hypothesis, keep the model class well-typed and
non-duplicative: a class of laws is constrained only by law-level conditions
(estimator-side conditions belong on the achievability theorem), and each
structural exponent is stated once — do not add a condition restating one already
present.
