=== UNIVERSAL PROOF DISCIPLINE (all four clusters) ===

Proof style:
- Theorem–proof style: definitions, assumptions, theorem statements, proof, remarks, sanity checks.
- The proof is natural language but equation-by-equation, so each step is self-contained and traceable to a named assumption or algebraic identity.
- Explicitly state all support, rank, positivity, overlap, and boundary conditions.

Causal interpretation:
- Distinguish the causal target from the identifying / regression / bounding functional. Do not collapse them by definition.
- Point identification (Panel / ExactID): rewrite the result as `target effect + contamination terms` whenever possible; do not claim identification unless the contamination vanishes under explicit assumptions.
- Partial identification: state the result as `Θ_I(P) ⊆ [θ_L(P), θ_U(P)]` with a sharpness verdict; if not sharp, flag it.
- Stat: identification of the causal estimand θ is GIVEN (cite the identifying functional; do not re-derive it). The result is about the *estimator's* behavior — a convergence rate, an efficiency bound, or a limit law — and must ADVANCE THE FRONTIER in a recognized way: (a) a matched pair (rate + a matching minimax lower bound, proving tightness); (b) a SHARPER bound than a named published result (strictly faster rate or smaller constant — state what it improves); (c) the FIRST nontrivial bound for a named method/estimand that lacked one; (d) an efficiency bound with an attaining estimator; or (e) a limit law. A matching lower bound PROVES tightness and is the strongest form, but is NOT the only creditable contribution — a strict improvement over prior art or a first nontrivial bound counts. Distinguish the estimator from its target and from the limit object.

Usability of the conclusion (UNIVERSAL): the theorem must expose its conclusion in a form a downstream reader can act on. For Panel / ExactID / PartialID the bar is identification / structure level — sampling, plug-in, inference are out of scope. **Stat and Experimentation are the exceptions: for Stat the rate / efficiency / limit-law IS the contribution; for Experimentation the design-based limit law + Wald-coverage (design given) OR the optimal design + its optimality certificate (design chosen) IS the contribution, derived from primitive design conditions and never assumed.** By theorem type:

- **Identifying-formula / bound theorems.** The named functional must be a *computable map* from observed-population inputs (the gate is evaluability, not any formalism). Acceptable forms include: closed-form functional of P; CDF/quantile/density envelope or copula identity; extremal-coupling / optimal-transport representation; explicit support-function / random-set kernel; explicit algorithm with stated complexity; reduction to a named optimization with explicit dimension/variables/constraints; regularized-inverse / completeness-bridge formula with stated regularization or basis. NOT acceptable: "θ = inf/sup of an unspecified set", "θ is identified" without the formula, a contamination term named but not written.
    - Panel — contamination terms are explicit projection/residual functionals.
    - ExactID — write the identifying formula (back-door / g-formula / Wald / front-door / mediation / proximal-bridge) explicitly; if it contains an inverse, state the algorithm/basis making it computable.
    - PartialID — bound endpoints expose evaluability in some explicit form (closed form, CDF/quantile/copula envelope, extremal coupling, support-function kernel, vertex characterization, or a named finite optimization). Bare inf/sup is not usable.
    - Stat — the estimator is an explicit map from the sample (M-estimator / plug-in / one-step / orthogonal-moment / sieve, written out); the target is a named rate `n^{-β}` (β tied to a named structural exponent — smoothness, overlap-decay, sparsity), an efficiency bound (semiparametric variance / influence function), or a limit law. The result must ADVANCE THE FRONTIER in one of (a matching minimax lower bound is NOT required): a matched lower bound (named two-point / Fano / Assouad / Le-Cam construction — proves tightness), a SHARPER upper bound than a named prior result, the FIRST nontrivial bound for a named method/estimand, attainment of the efficiency bound, a limit law, OR a nontrivial estimator (consistency + a rate or limit law, not necessarily sharp) for a NOVEL causal estimand the literature had not estimated. (A matched pair is strongest; a strict upper-bound improvement, a first nontrivial bound, or a novel-estimand estimator is also creditable — do not refuse to solve such a kernel because it lacks a converse.)
    - Experimentation — identification is GIVEN by the design (finite-population; randomness = assignment, outcomes fixed). For an inference result, the estimator is an explicit map from the realized assignment and the conclusion is a limit law + Wald-coverage DERIVED from primitive design / dependency conditions (never assumed). For an optimal-design result, write the feasible design class + criterion and exhibit the optimal design WITH an optimality certificate (KKT / matching variance lower bound / saddle), not a design merely asserted to be good.
- **Counterexample / negative-result theorems.** The obstruction is an explicit witness: a concrete observable law on which the failure occurs (failure quantified), or a generic-class certificate.
- **Equivalence / characterization theorems.** At least one side is a computable map, or both sides are named objects already known to be evaluable.
- **Pure impossibility / generic-class theorems.** The deliverable is the obstruction / genericity certificate (quantifier-elimination, dimension count, topological argument) — explicit, not "by a standard argument".

=== ASSUMPTION / DEFINITION HYGIENE ===

- Every definition, assumption, and theorem statement is unambiguous and well-typed: each symbol defined before use; one name per object, a class kept notationally distinct from its members; a defined set (model class, identified set) carved ONLY by conditions that are properties of its elements.
- Each assumption body states ONLY the mathematical condition it asserts. Motivation, "this is standard", where-used pointers, derived consequences ("thus …"), and sufficiency derivations belong in a remark or a separate lemma — NOT the assumption body. One assumption asserts one coherent condition (no unrelated measurability/integrability omnibus folded into a scale/rate assumption).
- A condition referenced by name (Lindeberg, Bernstein, Tsybakov margin, Hölder, …) is written out as its actual inequality/display at first use — a name alone, or a disjunction of named names, is not a statement.
- State every assumption used by a division, inverse, limit, conditioning step, rank argument, or case split. If a NEW assumption is needed to close a step, surface it explicitly (it must be standard + needed + natural) — never bury it mid-proof.
- Cite published references for known results (Backdoor: Pearl 1995; Manski bounds: Manski 1990; Balke–Pearl 1997; …).
