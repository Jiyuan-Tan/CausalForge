=== GLOBAL SETUP — Stat / estimation-and-inference anchor ===

What this cluster is (and is NOT):
- The contribution is the *estimation and inference theory* of a CAUSAL estimand — a minimax convergence rate, a semiparametric efficiency bound, a limit law / CLT, a finite-sample concentration result, or a coverage-facing inference theorem. The rate / efficiency / limit / coverage statement IS the headline.
- Identification is GIVEN, not the contribution. Cite the identifying functional for the estimand and take it as known. If your novelty is a new identification or a new bound, you are in the WRONG cluster (route to ExactID / PartialID with the estimator as the M8 rung) — do not re-anchor here.
- The estimand must be CAUSAL: ATE / ATT / CATE / LATE / policy value / a partial-ID bound endpoint / a dose-response or derivative-effect curve. A rate for an arbitrary statistical functional with no causal estimand is out of scope.

Target object + estimand structure:
- State the causal estimand θ and its identifying functional (given). State the statistical model / nuisance class (smoothness, sparsity, overlap, factor structure, mixing) and the sampling model (iid / panel / network draw; moment / tail conditions) precisely, each as a labeled assumption.
- DEFINE THE MODEL CLASS PRECISELY AND MINIMALLY. Write 𝒫 for the class, `P` for a member law. 𝒫 is carved only by conditions a single law can satisfy (law properties / functionals of the law); estimator- and inference-side conditions (nuisance-rate envelopes, cross-fit, empirical-process scale, tuning like clipping/trimming/sieve) are hypotheses of the achievability theorem on the estimator, NOT members of 𝒫. The same 𝒫 governs both sides of the converse: lower-bound instances must be members, the upper bound holds for every member.
- Define the estimator as an EXPLICIT map from the sample (M-estimator / plug-in / one-step / orthogonal-moment / cross-fit / sieve / trimmed form — written out, not "there exists an estimator").
- Define the target object: a convergence rate `n^{-β}` with β tied to a NAMED structural exponent; a semiparametric efficiency bound (the influence function / asymptotic variance); or a limit law.

The result must ADVANCE THE FRONTIER (this is the Stat floor) in one of:
- a matched pair — a rate paired with a minimax LOWER bound proving it unbeatable (named construction — two-point / fuzzy two-point / Le Cam, Fano / M-ary, Assouad / d-cube). This PROVES tightness and is the strongest form;
- a SHARPER bound than a named published result — a strictly faster rate or smaller constant (state what it improves);
- the FIRST nontrivial bound for a named method/estimand that lacked one;
- an efficiency bound paired with an ATTAINING estimator (or a lower bound that the estimator attains — state which direction each result delivers);
- a limit law.
A matched pair is strongest, but a sharper bound or a first nontrivial bound is itself a genuine contribution — do NOT treat a missing converse as disqualifying when the bound improves prior art or is the first for its method. A lone bound that is none of the above (not tight, not an improvement, not a first) caps at field.
- LOOSE-RATE FALLBACK (do this rather than leave a side open). If the MATCHED/sharp rate does not close — the converse and achievability exponents do not coincide, or one side resists a clean closed form — do NOT abandon the upper (or lower) side as merely OPEN. DERIVE the best obtainable LOOSE rate on that side and present the honest two-sided BRACKET `c n^{-r_lower} <= target <= C n^{-r_upper}`. The standard route for a loose achievability is to OPTIMIZE the estimator's own tuning (clip / trim / bandwidth / sieve dimension) by a bias–variance balance to the best honest exponent — a derived bracket that advances the frontier beats an OPEN gap. The achievable exponent may come out as a FUNCTION of the declared nuisance / smoothness rates (or as the value of that tuning optimization) rather than a single absolute constant: that parameterized exponent IS the closed result (the DML / structure-agnostic / semiparametric form) — commit to it, state its dependence on the auxiliary rates, and give the sub-regime where it meets the converse; do NOT mark such a side "open" just because its exponent is not a bare number. Quote what closes vs. what stays open: a genuinely loose bracket whose tightness is unknown is a valid result (record "is the bracket tight?" as the residual OEQ), and frequently the bracket is tight in a sub-regime (e.g. matches at the no-decay / regular endpoint) and loosens away from it — state that endpoint explicitly. NEVER strengthen an assumption to force the match (that is the laundering guard); the fallback is a BETTER-tuned estimator or a sharper proof under the SAME assumptions, or an honestly looser exponent.
- Distinguish the estimator, its target θ, and the limit object. Do not collapse them.

Proof technique:
- **Use any technique that fits the problem.** The catalogue below is a freely-ignorable suggestion menu:
    * lower bounds: Le Cam two-point / fuzzy hypotheses, Fano's inequality (M-ary), Assouad's cube, Ingster χ² / multiple-testing reductions;
    * upper bounds / limit laws: empirical-process / Donsker & stochastic-equicontinuity arguments, Z-/M-estimator expansions, Neyman-orthogonal / debiased (one-step) constructions with cross-fitting, U-statistic Hoeffding decomposition, sample-quantile Bahadur expansion;
    * efficiency: semiparametric efficiency-bound / tangent-space / influence-function calculus, Hájek–Le Cam convolution / LAN;
    * finite-sample: Hoeffding / Bernstein / sub-exponential concentration, empirical-Bernstein, McDiarmid;
    * structure-agnostic / DML rate calculus (product-of-nuisance-rate bias, critical radius).
- The only hard requirement: each step is traceable to a named assumption or a named theorem/identity, and whichever frontier-advance the result claims (a matching converse, a sharper bound vs named prior art, or a first nontrivial bound) is actually PROVED, not asserted.
