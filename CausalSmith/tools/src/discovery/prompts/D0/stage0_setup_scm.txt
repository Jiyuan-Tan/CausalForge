=== GLOBAL SETUP — SCM / graphical identification + partial identification anchor ===

What this cluster is (and is NOT):
- The contribution is a GRAPHICAL / STRUCTURAL result about a CAUSAL query stated over a structural causal model — a DAG / ADMG / selection diagram — in the interventional (do / σ-calculus) language. The cluster identity: the CAUSAL GRAPH is a first-class object, and the query, the assumptions, and the obstruction are all expressed graphically. If your result is stated purely in potential-outcomes / exchangeability language with NO graph, you are in the WRONG cluster (route to exactid / partialid).
- The query must be CAUSAL: an interventional distribution P(y | do(x)), a counterfactual (Layer-3) query, a path-specific / mediated effect, or a contrast thereof — expressed over the graph. A result about an arbitrary graph functional with no causal query is out of scope.
- TWO native lanes (both admissible; do NOT force a question into the wrong one):
    (1) **Identification under assumptions.** Give a NEW identifying functional Q = f(P_obs) for a causal query that holds under stated assumptions on (G, SCM class). The assumption family is OPEN — do not limit it to a fixed list. Common forms: extra structural constraints (monotonicity, shape / parametric), a harder query class (counterfactual / Layer-3), or auxiliary structure (proxies / negative controls, selection nodes, extra experiments / interventional inputs). A GENUINELY NOVEL assumption class introduced by the result itself is equally valid, and the novelty of the assumption can itself be the contribution (e.g. bridge-function / completeness conditions on a proxy pair). The identifying functional + its derivation IS the headline.
    (2) **Partial identification.** When the query is NOT point-identified, give a PRACTICAL, VALID bound Q ∈ [L(P_obs), U(P_obs)] with a method to compute / estimate it. Soundness (every SCM compatible with G and P_obs satisfies the bound) is required; the bound must be non-trivial (strictly inside the logical bounds). The valid bound + its soundness IS the headline. This includes data-fusion / selection-diagram bounds combining observational + interventional + selected data.

Target object + query structure:
- State the causal graph (DAG / ADMG / selection diagram) and the causal query precisely. State every assumption as a labeled hypothesis — the graph structure, any structural constraint (monotonicity / shape), and the available-distribution set (which observational / interventional / selected distributions are given).
- For an IDENTIFICATION result (lane 1): define the identifying functional as an EXPLICIT map Q = f(P_obs) (written out, not "there exists a functional"), and the target object as the theorem that Q equals that functional under the labeled assumptions. The proof is a direct derivation (do / σ-calculus manipulation, or an SCM-level argument) — analogous to a backdoor / frontdoor derivation under the extra assumption.
- For a PARTIAL-IDENTIFICATION result (lane 2): define the bound [L, U] as EXPLICIT maps of P_obs, and the target object as SOUNDNESS — every SCM in the compatible class (the SCMs consistent with G and the observed distributions) has Q ∈ [L, U]. The compatible class is made finite/tractable via the canonical / response-function reduction. State any estimation method (plug-in / LP / closed form) as part of the practical-method requirement.

The result must satisfy the SCM floor:
- **Lane 1 (identification) — `N-id-novelty`.** The identifying functional must be NEW: a new assumption set (including a novel assumption class), a new query, or a new graph / data configuration that yields identification — NOT a restatement of a known ID result (backdoor / frontdoor / classical do-calculus ID). COMPLETENESS IS NOT REQUIRED — most ID papers do not prove it — but when a result ALSO establishes completeness (the assumption is necessary, or the algorithm is complete for the query class) that is PREFERRED and scores higher. Do not gate on completeness.
- **Lane 2 (partial identification) — `N-pid-practical`.** The bound must be VALID (soundness proved from the compatible-SCM class) and PRACTICAL (computable / estimable) and NON-TRIVIAL (strictly inside the logical bounds). SHARPNESS IS NOT REQUIRED — a valid relaxed (non-tight) bound stands — but a SHARP bound (backed by compatible SCMs attaining L and U) is PREFERRED and scores higher. Do not gate on sharpness.
- The query stays CAUSAL and graph-native throughout; the assumptions stay explicit and labeled.

Proof technique:
- **Use any technique that fits the problem.** The catalogue below is a freely-ignorable suggestion menu:
    * identification: do-calculus (3 rules) and σ-calculus derivations, c-component / Q-factorization, the fixing operator, adjustment / back-door / front-door generalizations, proxy bridge-function arguments, transport / selection-diagram manipulation;
    * partial identification: the canonical / response-function SCM reduction (finite exogenous types), soundness over the compatible-SCM class, support-function / random-set calculus, linear / polynomial-program bounds (state the bound in closed form — the SOLVER is not formalized, the resulting bound + soundness is);
    * sharpness (optional): explicit construction of compatible SCMs attaining each endpoint.
- The only hard requirement: each step is traceable to a named assumption or a named theorem / identity, and the headline (identifying functional OR valid bound) is actually DERIVED (identification) or PROVED SOUND (partial ID) — not asserted.

Stress-test before committing a result (or declaring an open gap):
- Build the SMALLEST instance and check the claim EXACTLY; if a stress-test refutes the target AS STATED, narrow it honestly (per the solver's too-strong rule), never launder.
    * Identification lane: build a ≤4-node model satisfying the assumptions and verify the claimed functional equals the true query by direct computation; then build a model that VIOLATES one assumption and check the functional breaks — if it still holds, the assumption is not load-bearing and the "identification under that assumption" claim is mis-attributed.
    * Partial-ID lane: enumerate compatible SCMs on a small discrete model and check (a) every one lands inside [L, U] (soundness) and (b) the bound is strictly tighter than the logical bounds (non-triviality); if a claimed-sharp endpoint is not attained by any compatible SCM, downgrade the sharpness claim to a valid relaxed bound rather than overclaiming.
